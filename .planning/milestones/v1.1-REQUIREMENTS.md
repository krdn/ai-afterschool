- [x] **DEPLOY-02**: Caddy 리버스 프록시로 자동 SSL/TLS 인증서를 설정할 수 있다 — ✅ 완료 (Phase 8)
- [x] **DEPLOY-03**: `/api/health` 엔드포인트로 DB 연결 상태를 확인할 수 있다 — ✅ 완료 (Phase 8)
- [x] **DEPLOY-04**: `.dockerignore`로 환경변수 파일을 Docker 이미지에서 제외할 수 있다 — ✅ 완료 (Phase 8)
- [x] **DEPLOY-05**: 환경별 (dev/staging/prod) 설정 파일을 분리해서 관리할 수 있다 — ✅ 완료 (Phase 8)
- [x] **DEPLOY-06**: `prisma migrate deploy`로 데이터베이스 마이그레이션을 자동화할 수 있다 — ✅ 완료 (Phase 9)
- [x] **DEPLOY-07**: 무중단 배포 (zero-downtime) 배포 전략을 구현할 수 있다 — ✅ 완료 (Phase 8)

### Storage Migration (STORAGE)

- [x] **STORAGE-01**: MinIO S3 호환 스토리지를 Docker Compose로 실행할 수 있다 — ✅ 완료 (Phase 8)
- [x] **STORAGE-02**: PDF 저장 추상화 인터페이스를 구현할 수 있다 (로컬/MinIO 전환 가능) — ✅ 완료 (Phase 8)

## Out of Scope

명시적으로 제외된 기능입니다.

| Feature | Reason |
|---------|--------|
| Kubernetes 배포 | 단일 서버 배포에는 Docker Compose로 충분, 복잡도 과도 |
| 다중 리전 CDN | 단일 조직 배포, 비용 대비 효과 낮음 |
| 실시간 웹소켓 | 현재 요구사항 없음, 필요 시 v2+ 검토 |
| 마이크로서비스 아키텍처 | 모놀리식 Next.js로 충분, 분리 필요성 낮음 |
| 완전한 CI/CD 파이프라인 | v1.1은 수동 배포 자동화, GitHub Actions는 v2+ |
