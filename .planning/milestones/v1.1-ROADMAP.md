# Milestone v1.1: Production Readiness

**Status:** ✅ SHIPPED 2026-01-30
**Phases:** 8-10
**Total Plans:** 22

## Overview

v1.1 Production Readiness는 개발 환경을 프로덕션 배포용으로 변환하여 단일 조직(50~200명 학생)용 홈 서버에서 안정적으로 운영할 수 있도록 합니다. Docker 기반 배포 인프라, 데이터베이스 및 렌더링 성능 최적화, 기술 부채 해결(PDF 저장소 S3 호환 스토리지 마이그레이션, 코드 중복 제거), 프로덕션 모니터링을 포함합니다.

## Phases

### Phase 8: Production Infrastructure Foundation

**Goal**: Docker 기반 배포 인프라 구축 (SSL, S3 호환 PDF 저장소, 헬스 모니터링)
**Depends on**: v1.0 MVP (Phase 7)
**Requirements**: DEPLOY-01, DEPLOY-02, DEPLOY-03, DEPLOY-04, DEPLOY-05, DEPLOY-07, DEPLOY-08, STORAGE-01, STORAGE-02, STORAGE-03, STORAGE-04
**Plans**: 10 plans

**Plans:**

- [x] 08-01: Docker Compose 프로덕션 구성 (멀티스테이지 빌드, 헬스체크)
- [x] 08-02: Caddy 리버스 프록시와 자동 SSL
- [x] 08-03: MinIO S3 호환 스토리지 설정
- [x] 08-04: PDF 저장소 추상화 레이어
- [x] 08-05: PDF 데이터 마이그레이션 스크립트
- [x] 08-06: 헬스체크 엔드포인트
- [x] 08-07: 환경변수 관리 및 .dockerignore
- [x] 08-08: 무중단 배포 및 롤백 전략
- [x] 08-09: PDF Presigned URL 만료 문제 해결
- [x] 08-10: CI/CD 자동 롤백 구현

**Details:**

**Success Criteria (what must be TRUE):**
1. 단일 `docker compose up -d` 명령으로 프로덕션 서버에 배포 가능
2. HTTPS가 자동 구성되어 프로덕션 도메인용 유효한 SSL 인증서 발급
3. `/api/health` 엔드포인트가 healthy 상태를 반환하고 Docker 헬스체크에 응답
4. PDF 파일이 MinIO S3 호환 스토리지에 영구 볼륨으로 저장
5. 기존 로컬 PDF 파일이 MinIO로 데이터 손실 없이 이관
6. PDF 다운로드가 보안 presigned URL을 통해 작동
7. 환경변수가 적절히 격리됨 (Docker 이미지에 포함되지 않음)
8. 무중단 배포가 작동하며 배포 실패 시 자동 롤백 활성화

**Key Files:**
- Dockerfile.prod (멀티스테이지 빌드)
- docker-compose.prod.yml (postgres, minio, app, caddy 4개 서비스)
- Caddyfile (리버스 프록시, 자동 SSL)
- scripts/migrate-pdfs-to-s3.ts (PDF 마이그레이션)
- src/lib/storage/* (PDF 저장소 추상화)
- scripts/deploy.sh, scripts/rollback.sh
- .github/workflows/deploy.yml (CI/CD)

## Milestone Summary

**Decimal Phases:**
- None (정수 위상만 사용)

**Key Decisions:**
- Docker Compose 선택 (단일 서버 배포에 충분, 복잡도 낮음)
- MinIO S3 호환 스토리지 (셀프 호스팅, 비용 효율적)
- Caddy vs Nginx (자동 SSL 설정 간단)
- 연결 풀 제한 10 (Docker 환경 작은 규모)
- Prisma include로 N+1 해결 (단일 쿼리로 관계 데이터 로드)
- Sentry 오류 추적 (프로덕션 모니터링 필수)
- Pino 구조화된 로깅 (JSON 형식, 요청 ID 추적)
- PDF API 프록시 패턴 (Presigned URL 만료 문제 해결)

**Issues Resolved:**
- PDF 로컬 파일시스템 → MinIO S3 마이그레이션 완료
- Presigned URL 1시간 만료 → API 프록시 패턴으로 해결
- fetchReportData 중복 (164라인) → 공유 모듈로 추출
- N+1 쿼리 (7개) → Prisma include로 1개 쿼리로 최적화 (85% 감소)
- Phase 1 VERIFICATION.md 누락 → Retrospective verification 완료

**Issues Deferred (Non-blocking):**
- 2개 `<img>` 태그가 Next.js Image 미사용 (face-analysis-panel.tsx, student-image-tabs.tsx)
- 여러 컴포넌트에서 사용하지 않는 import 존재
- Health check degradation 자동 알림 (수동 모니터링 필요)

**Technical Debt Incurred:**
- Caddy 서비스 프로덕션에서 주석 처리 (기존 Nginx 사용 중)
- 자동 알림 시스템 미구현 (수동 모니터링으로 대체)
