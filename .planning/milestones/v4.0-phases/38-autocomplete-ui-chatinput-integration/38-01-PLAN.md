`src/hooks/use-mention.ts` 파일을 생성한다. 이 훅은 react-mentions-ts의 `<Mention data={...}>` prop에 전달할 async 함수와 관련 유틸리티를 제공한다.

**훅 구조:**

```typescript
"use client"

import { useRef, useCallback } from 'react'
import type { MentionDataItem } from 'react-mentions-ts'
import type { MentionSearchResponse } from '@/lib/chat/mention-types'
```

**MentionExtra 타입 (export):**
```typescript
export type MentionExtra = {
  type: 'student' | 'teacher' | 'team'
  sublabel: string
}
```

**useMention 훅 (export):**

반환 타입: `{ fetchMentions: (query: string) => Promise<MentionDataItem<MentionExtra>[]> }`

내부 구현:
- `latestQueryRef = useRef<string>('')` — stale 쿼리 감지용
- `abortRef = useRef<AbortController | null>(null)` — 이전 요청 취소용

`fetchMentions` 함수 (useCallback):
1. 최소 2자 미만이면 즉시 빈 배열 반환 (API 서버 최소 쿼리 길이 동일)
2. `latestQueryRef.current = query` 업데이트
3. `await new Promise(resolve => setTimeout(resolve, 200))` — 200ms 디바운스
4. `if (latestQueryRef.current !== query) return []` — stale 쿼리 감지
5. `abortRef.current?.abort()` — 이전 요청 취소
6. 새 AbortController 생성 및 `abortRef.current = controller` 할당
7. `fetch('/api/chat/mentions/search?q=' + encodeURIComponent(query), { signal: controller.signal })` 호출
8. 응답 JSON을 `MentionSearchResponse`로 파싱
9. flat 배열 변환 (학생 → 선생님 → 팀 순서):
   ```typescript
   return [
     ...data.students.map(s => ({
       id: `student:${s.id}`,
       display: s.name,
       type: 'student' as const,
       sublabel: s.sublabel,
     })),
     ...data.teachers.map(t => ({
       id: `teacher:${t.id}`,
       display: t.name,
       type: 'teacher' as const,
       sublabel: t.sublabel,
     })),
     ...data.teams.map(t => ({
       id: `team:${t.id}`,
       display: t.name,
       type: 'team' as const,
       sublabel: t.sublabel,
     })),
   ]
   ```
10. catch 블록: AbortError 포함 모든 에러 → 빈 배열 반환

**occurrencesToMentionItems 유틸리티 함수 (export):**

Submit 시 react-mentions-ts의 MentionOccurrence[]를 서버로 전송할 MentionItem[]으로 변환하는 순수 함수:
```typescript
import type { MentionItem, MentionType } from '@/lib/chat/mention-types'

export function occurrencesToMentionItems(
  mentions: Array<{ id: string | number }>
): MentionItem[] {
  const seen = new Set<string>()
  return mentions.flatMap(m => {
    const raw = String(m.id)
    if (seen.has(raw)) return []
    seen.add(raw)
    const colonIdx = raw.indexOf(':')
    const type = raw.slice(0, colonIdx) as MentionType
    const id = raw.slice(colonIdx + 1)
    return [{ type, id }]
  })
}
```
- 중복 제거: 같은 엔티티를 여러 번 멘션해도 한 번만 서버에 전달
- id 형식: `"student:abc123"` → `{ type: "student", id: "abc123" }`

**중요 설계 결정 (리서치 기반):**
- data prop에 **함수** 형태로 전달해야 react-mentions-ts의 getSubstringIndex 이중 필터링을 방지
- react-mentions-ts의 DataSource 타입은 `(query: string) => MaybePromise<MentionDataItem[]>` — 콜백 패턴이 아님
- IME 처리는 react-mentions-ts 내장 `_isComposing` 플래그가 자동 차단하므로 별도 처리 불필요 (MENT-05 충족)
