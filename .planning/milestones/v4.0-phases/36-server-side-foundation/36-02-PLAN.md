  <action>
  `src/lib/chat/mention-resolver.ts`를 생성한다. 핵심 함수: `resolveMentions(mentions, session)`

  **함수 시그니처:**
  ```typescript
  export async function resolveMentions(
    mentions: MentionItem[],
    session: { userId: string; role: TeacherRole; teamId: string | null }
  ): Promise<MentionResolutionResult>
  ```

  **구현 상세:**

  1. **멘션 타입별 분류**: student/teacher/team으로 분류하여 batch 조회 (N+1 방지)

  2. **학생 데이터 조회** (resolveStudents 내부 함수):
     - RBAC 적용: DIRECTOR가 아닌 경우 `session.teamId`와 학생의 `teamId` 비교
     - 기본정보: name, grade, school
     - 분석 결과 조회 (별도 쿼리): SajuAnalysis, NameAnalysis, MbtiAnalysis, FaceAnalysis, PalmAnalysis (subjectType='STUDENT', subjectId=studentId), VarkAnalysis, ZodiacAnalysis (studentId)
     - PersonalitySummary (studentId)
     - 최근 상담 노트 3건: CounselingSession (studentId, orderBy sessionDate desc, take 3)
     - 조회된 데이터를 구조화된 한국어 요약 텍스트로 변환

  3. **선생님 데이터 조회** (resolveTeachers 내부 함수):
     - RBAC 적용: DIRECTOR가 아닌 경우 `session.teamId`와 선생님의 `teamId` 비교
     - 기본정보: name, role, team
     - 분석 결과: SajuAnalysis, NameAnalysis, MbtiAnalysis, FaceAnalysis, PalmAnalysis (subjectType='TEACHER')
     - 담당 학생 이름 리스트: `db.student.findMany({ where: { teacherId }, select: { name: true } })`
     - 최근 상담 세션 3건

  4. **팀 데이터 조회** (resolveTeams 내부 함수):
     - RBAC: DIRECTOR가 아니면 자신의 teamId만 접근 가능
     - 팀 기본정보: name
     - 구성원: teachers count + names, students count + names
     - 통계: 평균 학년, 학생 수

  5. **RBAC 실패 처리**:
     - 접근 불가 시: `accessDenied: true`, contextData: '' (빈 문자열)
     - 알림 메시지 생성: "○○○님은 접근 권한이 없어 제외되었습니다"
     - `logAuditAction({ action: 'MENTION_ACCESS_DENIED', entityType, entityId, changes: { reason, teacherId } })` 호출

  6. **엔티티 요약 텍스트 형식** (한국어 구조화 키-값):
     학생 예시:
     ```
     이름: 홍길동 | 학년: 3학년 | 학교: ○○중학교
     [사주분석] (result.interpretation이 있으면 축약, 없으면 "분석 없음")
     [MBTI] ENFP - (interpretation 축약)
     [성명학] (interpretation 축약)
     [관상] (result 요약)
     [손금] (result 요약)
     [VARK] (varkType - interpretation 축약)
     [별자리] (zodiacSign - interpretation 축약)
     [AI종합] (personalitySummary.coreTraits 축약)
     [최근상담] YYYY-MM-DD: (summary 축약, 최대 3건)
     ```
     각 분석 항목의 interpretation/result에서 최대 100자까지 축약.
     분석이 없는 항목은 "[항목] 분석 없음" 으로 표기.

  7. **반환 형식**: MentionResolutionResult { resolved, metadata, accessDeniedMessages }
