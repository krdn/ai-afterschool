<task type="auto">
  <name>Task 2: GET /api/chat/mentions/search 라우트 핸들러 구현</name>
  <files>src/app/api/chat/mentions/search/route.ts</files>
  <action>
`src/app/api/chat/mentions/search/route.ts` 파일을 신규 생성한다. 기존 `/api/providers/route.ts` 패턴(verifySession + NextRequest.nextUrl.searchParams + NextResponse.json)을 따른다.

**Import:**
- `NextRequest`, `NextResponse` from `next/server`
- `verifySession` from `@/lib/dal`
- `db` from `@/lib/db`
- `MentionType`, `MentionSearchItem`, `MentionSearchResponse` from `@/lib/chat/mention-types`

**상수:**
- `RESULTS_PER_TYPE = 5` (타입별 최대 5건, 총 최대 15건)
- `ROLE_LABELS` 맵: `{ DIRECTOR: '원장', TEAM_LEADER: '팀장', MANAGER: '매니저', TEACHER: '선생님' }` — 선생님 서브레이블에서 한국어 역할명 표시용 (기존 `teacher-table.tsx`의 `roleLabels`와 일치)

**GET 핸들러 로직 (순서대로):**

1. **인증**: `const session = await verifySession()` — try 블록 안에서 호출. verifySession은 미인증 시 redirect()를 throw하며, Next.js가 이를 처리하므로 catch에서 별도 분기 불필요 (providers/route.ts 동일 패턴).

2. **쿼리 파라미터 파싱**:
   - `q = request.nextUrl.searchParams.get('q')?.trim() ?? ''`
   - `typesParam = request.nextUrl.searchParams.get('types') ?? ''`
   - `q.length < 1`이면 즉시 빈 응답 반환: `NextResponse.json({ students: [], teachers: [], teams: [] })`
   - types 파싱: `typesParam.split(',').map(t => t.trim()).filter(t => ['student','teacher','team'].includes(t))` — 빈 배열이면 3가지 모두 검색

3. **RBAC where 조건 빌더** (route.ts 내 헬퍼 함수 3개):
   - `buildStudentWhere(q, session)`: `{ name: { contains: q, mode: 'insensitive' } }` + DIRECTOR가 아니면 `{ teamId: session.teamId }` 추가. session.teamId가 null이면 결과 없음 보장 위해 `null` 반환 (호출측에서 빈 배열 반환).
   - `buildTeacherWhere(q, session)`: 동일 패턴. DIRECTOR가 아니면 `{ teamId: session.teamId }` 추가. session.teamId null이면 `null` 반환.
   - `buildTeamWhere(q, session)`: `{ name: { contains: q, mode: 'insensitive' } }` + DIRECTOR가 아니면 `{ id: session.teamId }` (팀은 id로 매칭). session.teamId null이면 `null` 반환.

4. **Promise.all 병렬 쿼리** (3개):
   - **학생**: `requestedTypes.includes('student')` 이고 where가 null이 아닐 때만 쿼리 실행. `db.student.findMany({ where, select: { id, name, grade, school, birthDate, images: { where: { type: 'profile' }, select: { resizedUrl }, take: 1 } }, orderBy: { name: 'asc' }, take: RESULTS_PER_TYPE })`. 조건 미충족 시 `Promise.resolve([])`.
   - **선생님**: `requestedTypes.includes('teacher')` 이고 where가 null이 아닐 때만 쿼리 실행. `db.teacher.findMany({ where, select: { id, name, role, profileImage, _count: { select: { students: true } } }, orderBy: { name: 'asc' }, take: RESULTS_PER_TYPE })`.
   - **학급**: `requestedTypes.includes('team')` 이고 where가 null이 아닐 때만 쿼리 실행. `db.team.findMany({ where, select: { id, name, _count: { select: { students: true, teachers: true } } }, orderBy: { name: 'asc' }, take: RESULTS_PER_TYPE })`.

5. **응답 변환** (DB 결과 → MentionSearchItem[]):
   - 학생: `sublabel = "${grade}학년 · ${school} · ${birthDate.toISOString().slice(0,10)}"`, `avatarUrl = images[0]?.resizedUrl ?? null`
   - 선생님: `sublabel = "${ROLE_LABELS[role] ?? role} · 담당 ${_count.students}명"`, `avatarUrl = profileImage ?? null`
   - 학급: `sublabel = "학생 ${_count.students}명 · 교사 ${_count.teachers}명"`, `avatarUrl = null`

6. **응답**: `NextResponse.json({ students, teachers, teams } satisfies MentionSearchResponse)`

7. **에러 처리**: catch 블록에서 `console.error('[MentionSearch] Error:', error)` 후 `NextResponse.json({ error: 'Internal server error' }, { status: 500 })`.

**하지 않을 것:**
- `export const dynamic = 'force-static'` 추가하지 않음 (RBAC 결과는 사용자별 다름)
- `getSession` 사용하지 않음 (`verifySession`이 신규 라우트 표준)
- 서버 측 AbortController 처리 불필요 (클라이언트 전용 — Phase 38에서 fetch abort 구현)
- `'server-only'` import 불필요 (route.ts는 서버 전용이 보장됨)
