# Plan 17-04: 상태 전환 및 CounselingSession 연결

## 목표
예약 상태를 전환하고 COMPLETED 시 CounselingSession을 자동 생성합니다.

## 작업

### Task 1: 상태 전환 스키마 작성
**파일:** `src/lib/validations/reservations.ts`

1. `statusTransitionSchema` - reservationId, newStatus (COMPLETED/CANCELLED/NO_SHOW)
2. `completeReservationSchema` - summary (선택적 상담 내용)

### Task 2: 데이터베이스 함수 작성
**파일:** `src/lib/db/reservations.ts`

1. `transitionReservationStatus()` - 상태 전환
   - 현재 상태가 SCHEDULED인지 확인
   - COMPLETED 시 CounselingSession 생성
   - 트랜잭션으로 원자적 처리
   - counselingSessionId 연결

2. CounselingSession 생성 로직
   - studentId, teacherId, sessionDate (예약 시간)
   - duration: 30 (기본값)
   - type: "ACADEMIC" (기본값, topic에서 변환 가능)
   - summary: 빈 값 또는 입력값

### Task 3: Server Actions 작성
**파일:** `src/lib/actions/reservations.ts`

1. `completeReservationAction()` - 완료 전환
   - 인증 및 권한 체크
   - 트랜잭션으로 상태 변경 + 세션 생성
   - revalidatePath()

2. `cancelReservationAction()` - 취소 전환
   - 인증 및 권한 체크
   - 상태를 CANCELLED로 변경
   - revalidatePath()

3. `markNoShowAction()` - 불참 전환
   - 인증 및 권한 체크
   - 상태를 NO_SHOW로 변경
   - revalidatePath()
