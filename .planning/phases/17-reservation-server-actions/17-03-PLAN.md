# Plan 17-03: 예약 수정 및 삭제

## 목표
SCHEDULED 상태의 예약을 수정하고 삭제하는 기능을 구현합니다.

## 작업

### Task 1: 수정 스키마 작성
**파일:** `src/lib/validations/reservations.ts`

1. `reservationUpdateSchema` - 모든 필드 선택적
2. `reservationDeleteSchema` - reservationId 필수

### Task 2: 데이터베이스 함수 작성
**파일:** `src/lib/db/reservations.ts`

1. `updateReservation()` - 예약 수정
   - 현재 상태가 SCHEDULED인지 확인
   - 시간 변경 시 중복 검증
   - 트랜잭션으로 원자적 처리

2. `deleteReservation()` - 예약 삭제
   - SCHEDULED 상태만 삭제 가능
   - soft delete 또는 hard delete (Prisma cascade 고려)

### Task 3: Server Actions 작성
**파일:** `src/lib/actions/reservations.ts`

1. `updateReservationAction()` - 수정
   - 인증 및 권한 체크
   - Zod 검증
   - RBAC 체크
   - updateReservation() 호출
   - revalidatePath()

2. `deleteReservationAction()` - 삭제
   - 인증 및 권한 체크
   - 상태 검증
   - 삭제 처리
   - revalidatePath()

## 검증 기준
- [ ] SCHEDULED 상태 예약만 수정 가능
- [ ] COMPLETED/CANCELLED/NO 상태 수정 시 "이미 완료된 예약은 수정할 수 없습니다" 에러
- [ ] 시간 변경 시 중복 검증 동작
- [ ] 삭제 후 예약이 목록에서 제거됨
