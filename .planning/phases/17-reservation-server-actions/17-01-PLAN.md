# Plan 17-01: 예약 생성 Server Actions 구현

## 목표
상담 예약 생성 기능을 Server Actions로 구현하고, 중복 검증과 RBAC를 적용합니다.

## 작업

### Task 1: Zod 검증 스키마 작성
**파일:** `src/lib/validations/reservations.ts`

1. 30분 단위 시간 슬롯 검증 (분이 0 또는 30인지 확인)
2. 필수 필드 검증: scheduledAt, studentId, parentId, topic
3. 한국어 에러 메시지

### Task 2: 데이터베이스 레이어 함수 작성
**파일:** `src/lib/db/reservations.ts`

1. `createReservationWithConflictCheck()` - 트랜잭션 기반 중복 검증
   - 같은 선생님의 같은 시간대 예약 확인 (CANCELLED 제외)
   - 충돌 시 에러 발생
   - 충돌 없으면 예약 생성

### Task 3: Server Action 작성
**파일:** `src/lib/actions/reservations.ts`

1. `createReservationAction()` - 예약 생성
   - verifySession() 인증 체크
   - Zod 스키마 검증
   - TEACHER 역할 시 RBAC 체크 (getRBACPrisma)
   - createReservationWithConflictCheck() 호출
   - revalidatePath()로 캐시 무효화
   - { success, error? } 형식 응답

## 검증 기준
- [ ] 30분 단위가 아닌 시간으로 예약 시 거부됨
- [ ] 같은 선생님의 같은 시간에 예약 시 "이미 예약이 있습니다" 에러
- [ ] 다른 선생님의 같은 시간은 예약 가능
- [ ] CANCELLED 상태 예약의 시간대는 재예약 가능
- [ ] TEACHER가 다른 팀 학생 예약 시 권한 에러
