의심스러운 비밀번호 재설정 시도를 로깅합니다:

1. 로깅 조건 정의:
- 유효하지 않은 토큰으로 반복 접근 (같은 IP/사용자)
- 만료된 토큰으로 다시 시도
- 이미 사용된 토큰으로 반복 시도

2. 로깅 구현 (Server Component):
```typescript
import { logSystemAction } from '@/lib/dal'
import { headers } from 'next/headers'

// 유효하지 않은 토큰인 경우 로깅 고려
if (!resetToken) {
  const headersList = await headers()
  const ip = headersList.get('x-forwarded-for') || headersList.get('x-real-ip') || 'unknown'

  // 주의: 모든 실패를 로깅하면 너무 많은 로그가 생성됨
  // 실제 환경에서는 rate limiting 후 로깅 권장
  await logSystemAction({
    level: 'WARN',
    message: 'Invalid password reset token accessed',
    context: { token, ip },
  })
}
```

3. 로깅 레벨 결정:
- invalid 토큰: WARN (의심스러운 활동)
- expired 토큰: INFO (일반적인 만료)
- used 토큰: INFO (정상적인 흐름)

주의:
- 개인정보 보호를 위해 IP 주소를 부분적으로 마스킹 고려
- 과도한 로깅은 성능에 영향을 줄 수 있음
- 실제 운영에서는 rate limiting과 결합 권장
