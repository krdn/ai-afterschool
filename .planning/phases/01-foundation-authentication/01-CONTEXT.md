# Phase 1: Foundation & Authentication - Context

**Gathered:** 2026-01-27
**Status:** Ready for planning

<domain>
## Phase Boundary

선생님이 학생 정보를 안전하게 등록하고 관리할 수 있다. 이메일/비밀번호 기반 인증으로 선생님 계정을 보호하고, 학생 기본 정보(이름, 생년월일, 연락처, 학교, 학년, 목표 대학/학과, 혈액형)를 등록/조회/검색할 수 있다.

**범위 내:**
- 선생님 로그인/로그아웃
- 비밀번호 재설정
- 학생 CRUD (생성, 조회, 수정, 삭제)
- 학생 목록 검색

**범위 외:**
- 학생 사진 업로드 (Phase 2)
- 분석 기능 (Phase 3+)

</domain>

<decisions>
## Implementation Decisions

### 로그인/인증 흐름
- 미니멀 중앙 정렬 레이아웃: 흰 배경에 로그인 카드만 중앙 배치
- 친절하고 구체적인 에러 메시지: "이메일 또는 비밀번호가 일치하지 않아요. 다시 확인해주세요." 식으로 존댓말 사용
- 세션 만료 시 자동 로그아웃 + "오래 활동이 없어 로그아웃되었습니다" 안내 후 로그인 페이지로 이동
- 비밀번호 재설정: 이메일 링크 방식 (임시 비밀번호 아닌 재설정 페이지 링크 발송)

### 학생 목록 UI
- 테이블 형식: 행 단위로 학생 표시, 엑셀처럼 정렬/필터 용이
- 핵심 정보만 표시: 이름, 학교, 학년, 목표대학. 상세는 클릭 시 확인
- 통합 검색박: 한 개의 검색창에서 이름, 학교 동시 검색
- 페이지 번호 방식 페이지네이션: 10명/20명/50명 단위 선택 가능

### 학생 등록 폼
- 단일 페이지 폼: 모든 필드를 한 화면에 배치 (필드 수가 적어 단계 불필요)
- 필수 필드만 별표(*) 표시, 선택 필드는 별도 표시 없음
- 실시간 인라인 유효성 검증: 입력 중 오류 즉시 필드 아래에 표시
- 등록 완료 후 해당 학생 상세 페이지로 이동

### 빈 상태/온보딩
- 학생이 없을 때: 친근한 일러스트 + "학생을 등록해보세요!" 안내 + "첫 학생 등록하기" 버튼
- 온보딩 투어 없음: UI가 직관적이므로 별도 투어 불필요
- 버튼/아이콘에 호버 툴팁 제공: 마우스 올리면 간단한 설명 표시
- 에러 발생 시 토스트 알림: 화면 상단에 일시적으로 표시 후 자동 사라짐

### Claude's Discretion
- 로딩 스켈레톤 디자인
- 정확한 색상/타이포그래피
- 토스트 알림 지속 시간
- 빈 상태 일러스트 스타일

</decisions>

<specifics>
## Specific Ideas

- 로그인 실패 메시지는 보안을 위해 "이메일 또는 비밀번호"로 통합 (어떤 것이 틀렸는지 노출하지 않음)
- 테이블은 깔끔하고 현대적인 느낌 (Linear, Notion 스타일 참고)
- 한국어 서비스이므로 모든 UI 텍스트는 자연스러운 한국어로

</specifics>
