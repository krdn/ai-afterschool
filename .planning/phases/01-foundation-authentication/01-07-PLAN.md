    - test@afterschool.com / test1234 로그인 → /students로 이동 확인

    **2. 세션 지속성 테스트 (AUTH-03)**
    - 로그인 상태에서 브라우저 새로고침 → 로그인 유지 확인
    - 새 탭에서 /students 접속 → 로그인 유지 확인

    **3. 다중 계정 테스트 (AUTH-04)**
    - 두 번째 선생님 계정 생성 (seed 또는 DB 직접)
    - 시크릿 창에서 두 번째 계정 로그인
    - 각 계정이 독립적으로 동작 확인

    **4. 비밀번호 재설정 테스트 (AUTH-02)**
    - /reset-password 접속
    - 이메일 입력 후 제출 → 성공 메시지 확인
    - (Resend 설정 시) 이메일 수신 → 링크 클릭 → 새 비밀번호 설정 → 로그인 성공

    **5. 학생 등록 테스트 (STUD-01, STUD-03, STUD-07)**
    - /students/new 접속
    - 필수 필드 누락 시 에러 메시지 확인
    - 모든 필드 입력 후 등록 → 상세 페이지로 이동 확인
    - 등록 정보(이름, 생년월일, 연락처, 학교, 학년, 목표 대학/학과, 혈액형) 확인

    **6. 학생 조회/검색 테스트 (STUD-04, STUD-05)**
    - /students 접속 → 테이블에 학생 목록 표시
    - 검색창에 학생 이름 입력 → 필터링 동작
    - 검색창에 학교명 입력 → 필터링 동작
    - 이름/학교 열 클릭 → 정렬 동작
    - 페이지 사이즈 변경 → 표시 개수 변경

    **7. 학생 상세 조회 테스트 (STUD-06)**
    - 학생 이름 또는 "상세보기" 클릭 → 상세 페이지
    - 모든 정보가 올바르게 표시되는지 확인

    **8. 학생 수정/삭제 테스트**
    - 상세 페이지에서 "수정" 클릭 → 수정 폼
    - 정보 변경 후 저장 → 상세 페이지에서 변경 확인
    - "삭제" 클릭 → 확인 후 삭제 → 목록에서 제거됨

    **9. 로그아웃 테스트**
    - 로그아웃 버튼 클릭 → /login으로 이동
    - /students 접속 시도 → /login으로 리다이렉트

    **10. 권한 분리 테스트**
    - 선생님 A로 학생 등록
    - 선생님 B로 로그인 → 선생님 A의 학생이 보이지 않음 확인
    - 선생님 A 학생의 URL 직접 접근 시 404 또는 접근 불가

    **11. UI/UX 확인**
    - 로그인 페이지: 미니멀 중앙 카드 레이아웃
    - 에러 메시지: 친절한 한국어 존댓말
    - 빈 상태: 학생이 없을 때 안내 메시지 + "첫 학생 등록하기" 버튼
    - 테이블: 깔끔한 디자인, 호버 상태
