<task type="auto">
  <name>Greedy 자동 배정 알고리즘 구현</name>
  <files>src/lib/optimization/auto-assignment.ts</files>
  <action>
1. src/lib/optimization/auto-assignment.ts 생성:

2. 타입 정의:
   - Assignment: { studentId, teacherId, score: CompatibilityScore }
   - AutoAssignmentOptions: { maxStudentsPerTeacher?, minCompatibilityThreshold?, teamId? }

3. generateAutoAssignment(studentIds, options) 함수:
   - verifySession()으로 인증 및 RBAC 확인
   - Teacher 목록 조회 (role in [TEACHER, MANAGER, TEAM_LEADER]):
     * include: teacherMbtiAnalysis, teacherSajuAnalysis, teacherNameAnalysis
     * include: _count { select { students: true } } (현재 부하)
   - Student 목록 조회 (studentIds in):
     * include: mbtiAnalysis, sajuAnalysis, nameAnalysis
   - 평균 부하 계산: averageLoad = totalStudents / totalTeachers
   - maxLoad = options.maxStudentsPerTeacher ?? Math.ceil(averageLoad * 1.2)
   - teacherLoads Map 초기화 (teacherId → 현재 학생 수)

   - 각 Student에 대해:
     * 모든 Teacher 순회하며 calculateCompatibilityScore 호출
     * currentLoad >= maxLoad인 Teacher는 건너뜀
     * minCompatibilityThreshold 미만인 점수는 건너뜀
     * 최고 점수 Teacher 선택 (Greedy)
     * assignments에 추가
     * teacherLoads 업데이트

   - assignments 반환

참고: RESEARCH.md의 Greedy algorithm 슈도코드 참조 (O(students × teachers) 복잡도)
  </action>
