  <action>
1. src/lib/analysis/fairness-metrics.ts 생성:

2. 타입 정의:
   - FairnessMetrics: { disparityIndex: number, abroca: number, distributionBalance: number, recommendations: string[] }

3. calculateFairnessMetrics(assignments) 함수:
   - assignments: Array<{ studentId, teacherId, score }>
   - calculateDisparityIndex(assignments) 호출
   - calculateABROCA(assignments) 호출
   - calculateDistributionBalance(assignments) 호출
   - generateFairnessRecommendations({ disparityIndex, abroca, distributionBalance }) 호출
   - FairnessMetrics 반환

4. calculateDisparityIndex(assignments) 함수:
   - db.student.findUnique로 각 Student의 school, grade 조회
   - 학교별 점수 그룹화 (schoolGroups Map)
   - 학교별 평균 점수 계산
   - 최대-최소 차이 (정규화: (max - min) / 100)
   - disparityIndex 반환 (0-1)

5. calculateABROCA(assignments) 함수:
   - 점수 분포를 히스토그램으로 변환 (bins = 10)
   - 이상적인 분포 (균등)과 L1 distance 계산
   - 정규화 (diff / assignments.length)
   - abroca 반환 (0-1)

6. calculateDistributionBalance(assignments) 함수:
   - Teacher별 배정 수 계산 (teacherCounts Map)
   - 평균(mean)과 표준편차(stdDev) 계산
   - 정규화 (1 - stdDev / mean)
   - distributionBalance 반환 (0-1)

7. generateFairnessRecommendations(metrics) 함수:
   - disparityIndex > 0.2: "학교 간 궁합 점수 차이가 큽니다. 가중치 재조정을 검토하세요."
   - abroca > 0.3: "궁합 점수 분포가 편향되어 있습니다. 알고리즘 검토가 필요합니다."
   - distributionBalance < 0.7: "선생님별 배정 분포가 불균형합니다. 부하 분산 가중치를 높이세요."
   - 추천 없음: "공정성 메트릭이 정상 범위입니다."

참고: RESEARCH.md의 Fairness Metrics Implementation 예시 코드 참조
