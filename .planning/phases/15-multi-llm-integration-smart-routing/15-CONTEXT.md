# Phase 15: Multi-LLM Integration & Smart Routing - Context

**Gathered:** 2026-01-31
**Status:** Ready for planning

<domain>
## Phase Boundary

다중 LLM 제공자 통합 및 비용 최적화 라우팅 시스템을 구축합니다. Ollama(로컬), Gemini, ChatGPT, Claude 중에서 LLM을 선택할 수 있게 하고, 자동 failover와 비용 기반 스마트 라우팅을 구현합니다. 토큰 사용량과 비용을 추적하고 예산 임계값 초과 시 알림을 제공합니다.

</domain>

<decisions>
## Implementation Decisions

### Admin 설정 UI

- **제공자 선택 인터페이스**: 제공자별 카드 형태 (Ollama, Gemini, ChatGPT, Claude 각각을 카드로 표시하고, 카드별로 API 키 입력 및 활성화/비활성화)
- **API 키 관리**: 입력 후 즉시 검증 — 사용자가 API 키를 입력하면 즉시 테스트 요청을 보내서 유효성 확인
- **API 키 표시**: 마스킹 처리 — `sk-***...***xyz` 형태로 앞뒤 몇 글자만 보여주고 중간은 별표
- **기본 모델 선택**: 기능별 모델 매핑 — 학습 분석은 A 모델, 상담 제안은 B 모델 등 기능별로 다르게 설정

### 라우팅 정책

- **우선순위 결정**: 기능별 매핑 — 각 기능(학습 분석, 상담, 보고서 등)마다 우선 제공자를 따로 지정
- **Failover 발동 시점**: 오류 즉시 — API 오류가 발생하면 즉시 다음 제공자로 시도
- **재시도 로직**: 모든 제공자 1회씩 — 우선순위대로 각 제공자를 1번씩 시도, 모두 실패하면 오류 반환
- **모든 제공자 실패 시**: 오류 반환 — 사용자에게 '일시적 오류, 나중에 다시 시도해주세요' 메시지

### 사용량 모니터링

- **추적 단위**: 모든 차원 — 기능, 사용자, 제공자, 모델 모두를 차원으로 기록하여 다차원 분석 가능
- **데이터 보관 기간**: 1년간 보관, 월별 집계 데이터는 영구 보관
- **집계 방식**: 실시간 — LLM 요청마다 DB에 즉시 기록, 대시보드는 항상 최신 데이터
- **추적 메트릭**: 전체 메트릭 — 비용, 성능, 요청 크기, 응답 크기, 제공자별 사용률 모두 기록
  - 토큰 사용량 (input/output)
  - 비용
  - 응답 시간
  - 성공/실패율
  - Failover 횟수
  - 요청/응답 크기
  - 제공자별 사용률

### 비용 알림

- **알림 조건**: 임계값 초과 — 일일/주간/월간 비용 예산을 설정하고, 80%/100% 도달 시 알림
- **알림 대상**: 원장만 — DIRECTOR 권한을 가진 사용자에게만 알림
- **알림 방식**: 인앱 알림 — 시스템 내 알림 센터에만 표시
- **알림 내용**: 링크 포함 — 간단한 메시지 + 비용 대시보드 링크

### Claude's Discretion

- 카드 디자인 상세 (색상, 레이아웃, 간격)
- API 키 검증 타임아웃 설정
- DB 스키마 구조 (LLMUsage, LLMConfig 모델)
- 대시보드 차트 라이브러리 선택 (Recharts 재사용 가능)
- 알림 센터 UI 구현 방식
