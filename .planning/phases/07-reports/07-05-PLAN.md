<task type="checkpoint:human-verify" gate="blocking">
  <what-built>
    Complete PDF report generation system:
    - Database schema with ReportPDF table
    - Korean font support (Noto Sans KR TTF)
    - PDF template components (7 files)
    - Server Actions for async generation
    - API endpoints for streaming and status
    - Report button UI with polling
    - Integration with student detail page
  </what-built>
  <how-to-verify>
    **Preparation:**
    1. Ensure dev server is running: `npm run dev`
    2. Ensure database is accessible and has test students
    3. Open browser to http://localhost:3000

    **Test Scenario 1: First-time PDF Generation**
    1. Login as teacher
    2. Navigate to a student detail page (with at least 3 completed analyses)
    3. Scroll to "상담 보고서" section
    4. Click "보고서 생성" button
    5. Verify button changes to "생성 중..." with spinner
    6. Wait for button to change to "PDF 다운로드" (should take 5-15 seconds)
    7. Click "PDF 다운로드"
    8. Verify PDF file downloads
    9. Open PDF and verify:
       - Korean text displays correctly (no squares/garbled)
       - Student info is complete and accurate
       - All analysis results are present (MBTI, Saju, Name, Face, Palm)
       - AI recommendations are shown (learning strategy, career guidance)
       - Layout is professional and readable
       - Footer has disclaimers and date

    **Test Scenario 2: Cached PDF (Second Generation)**
    1. On the same student page, click "보고서 생성" again
    2. Verify message: "이미 최신 보고서가 있습니다."
    3. Verify button immediately shows "PDF 다운로드" (no generating state)
    4. Download and verify it's the same PDF

    **Test Scenario 3: Status Persistence (Page Refresh)**
    1. While PDF is generating, refresh the page
    2. Verify button still shows "생성 중..." (status persisted)
    3. Wait for completion and download PDF

    **Test Scenario 4: Student with No Analyses**
    1. Navigate to a student with 0-2 completed analyses
    2. Scroll to "상담 보고서" section
    3. Click "보고서 생성"
    4. Verify PDF still generates (shows available data)
    5. Open PDF and verify partial data is handled gracefully

    **Test Scenario 5: Error Handling**
    1. Check browser console for errors
    2. Verify no TypeScript errors
    3. Test with invalid student ID (should show 404 or error)

    **Additional Checks:**
    - Verify button styling matches other components
    - Check responsive design on mobile
    - Verify polling stops after completion
    - Check for memory leaks (repeated generate/cancel cycles)
  </how-to-verify>
  <resume-signal>
    Type "approved" if all tests pass, or describe issues found (e.g., "Korean text shows as squares", "Generation doesn't complete", "PDF layout broken")
  </resume-signal>
</task>
