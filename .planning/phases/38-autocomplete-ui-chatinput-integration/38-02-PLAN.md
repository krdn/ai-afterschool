**5. handleSubmit 수정:**

const handleSubmit = useCallback(() => {
  // react-mentions-ts 마크업에서 plain text 추출: @[이름](type:id) → @이름
  const plainText = mentionMarkup.replace(/@\[([^\]]+)\]\([^)]+\)/g, '@$1').trim()

  const mentionItems = occurrencesToMentionItems(activeMentions)
  onSend(plainText, mentionItems, parseProviderId(selectedModel))

**6. handleKeyDown 수정 (Enter 키 충돌 방지):**
react-mentions-ts는 드롭다운이 열려있을 때 Enter를 가로채서 항목 선택을 처리하고, 닫혀있을 때만 외부 onKeyDown이 호출된다. 따라서 기존 Enter 핸들러 그대로 유지하면 된다.

```typescript
const handleKeyDown = useCallback(
  (e: React.KeyboardEvent) => {
    if (e.key === 'Enter' && !e.shiftKey) {
      e.preventDefault()
      handleSubmit()
    }
  },
  [handleSubmit]
)
```

**7. renderSuggestion 구현 (그룹 헤더 + Slack 스타일):**

그룹 헤더를 renderSuggestion 내에서 index 기반으로 주입한다.

```typescript
const groupLabels: Record<string, string> = {
  student: '학생',
  teacher: '선생님',
  team: '학급',
}

const renderSuggestion = useCallback((
  entry: SuggestionDataItem<MentionExtra>,
  _search: string,
  highlightedDisplay: React.ReactNode,
  index: number,
  focused: boolean
) => {
  // 이전 항목과 type이 다르면 그룹 헤더 삽입
  // react-mentions-ts의 renderSuggestion은 이미 suggestions 배열의 index를 순서대로 전달
  // data 함수가 학생→선생님→팀 순서로 flat 배열 반환하므로 type 전환 시 헤더
  const currentType = (entry as SuggestionDataItem<MentionExtra>).type

  return (
    <div>
      <div className={`flex items-center gap-2 px-3 py-1.5 ${focused ? 'bg-accent' : ''}`}>
        <div className="flex flex-col min-w-0">
          <span className="text-sm font-medium truncate">{highlightedDisplay}</span>
          {(entry as SuggestionDataItem<MentionExtra>).sublabel && (
            <span className="text-xs text-muted-foreground truncate">
              {(entry as SuggestionDataItem<MentionExtra>).sublabel}
            </span>
          )}
        </div>
      </div>
    </div>
  )
}, [])
```

**중요:** react-mentions-ts의 `renderSuggestion` index 파라미터는 현재 suggestion 배열 내 인덱스를 나타낸다. 그룹 헤더는 `customSuggestionsContainer`와 조합하여 처리하는 것이 더 안정적일 수 있다. 아래 접근법을 사용한다:

**customSuggestionsContainer 패턴 (PREFERRED):**

`<MentionsInput customSuggestionsContainer={...}>` prop을 활용하여 전체 suggestion 목록을 래핑하는 컨테이너를 커스터마이즈한다. 하지만 이 prop은 children을 받아 래핑만 하므로 개별 항목 사이에 헤더를 삽입할 수는 없다.

**최종 접근: renderSuggestion 내 그룹 헤더 주입:**

`fetchMentions`가 반환하는 flat 배열에 특별한 "헤더 아이템"을 삽입하는 대신, renderSuggestion에서 ref로 이전 타입을 추적한다:

```typescript
const prevTypeRef = useRef<string | null>(null)

// renderSuggestion이 호출될 때마다 index === 0이면 prevType 리셋
const renderSuggestion = useCallback((
  entry: SuggestionDataItem<MentionExtra>,
  _search: string,
  highlightedDisplay: React.ReactNode,
  index: number,
  focused: boolean
) => {
  const currentType = entry.type
  const showHeader = index === 0 || prevTypeRef.current !== currentType
  prevTypeRef.current = currentType

  // index === 0이면 리스트 시작이므로 항상 헤더 표시

  return (
    <div>
      {showHeader && (
        <div className="px-3 pt-2 pb-1 text-xs font-semibold text-muted-foreground uppercase tracking-wide border-t first:border-t-0 border-border">
          {groupLabels[currentType] || currentType}
        </div>
      )}
      <div className={`flex items-center gap-2 px-3 py-1.5 cursor-pointer ${focused ? 'bg-accent text-accent-foreground' : 'hover:bg-muted'}`}>
        <div className="flex flex-col min-w-0">
          <span className="text-sm font-medium truncate">{highlightedDisplay}</span>
          {entry.sublabel && (
            <span className="text-xs text-muted-foreground truncate">
              {entry.sublabel}
            </span>
          )}
        </div>
      </div>
    </div>
  )
}, [])
```

**8. JSX 교체 — Textarea → MentionsInput:**

기존 `<Textarea>` 부분을 교체:

```tsx
<MentionsInput
  value={mentionMarkup}
  onMentionsChange={handleMentionsChange}
  onKeyDown={handleKeyDown}
  placeholder={t("placeholder")}
  a11ySuggestionsListLabel="멘션 검색 결과"
  suggestionsPlacement="above"
  autoResize
  disabled={isStreaming}
  className="min-h-[44px] max-h-[160px] text-sm flex-1"
>
  <Mention
    trigger="@"
    data={fetchMentions}
    renderSuggestion={renderSuggestion}
    displayTransform={(_id: string, display: string) => `@${display}`}
    appendSpaceOnAdd
    markup="@[__display__](__id__)"
  />
</MentionsInput>
```

**핵심 설정:**
- `suggestionsPlacement="above"` — 채팅 입력창이 화면 하단이므로 드롭다운을 위로 표시 (RESEARCH Pitfall #6)
- `autoResize` — 텍스트 양에 따라 textarea 높이 자동 조절
- `appendSpaceOnAdd` — 멘션 선택 후 공백 자동 삽입
- `markup="@[__display__](__id__)"` — 내부 마크업 형식
- `trigger="@"` — 단일 trigger만 사용 (RESEARCH Pitfall #1: 중복 trigger 금지)
- `data={fetchMentions}` — 함수 형태 (RESEARCH Pitfall #3: 이중 필터링 방지)
- 키보드 네비게이션(↑↓ Enter Esc)은 react-mentions-ts 내장
- IME 차단(_isComposing)은 react-mentions-ts 내장
