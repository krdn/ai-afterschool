  <action>
    GradeHistory, CounselingSession, StudentSatisfaction 모델을 추가합니다.
    
    **GradeHistory 모델:**
    - id (UUID, PK)
    - studentId (UUID, FK to Student)
    - teacherId (String, FK to Teacher, nullable - 배정 변경 대응)
    - subject (String) - 과목명 (수학, 영어, 국어 등)
    - gradeType (Enum: MIDTERM, FINAL, QUIZ, ASSIGNMENT) - 시험/과제 유형
    - score (Float) - 점수
    - maxScore (Float) - 만점 (default 100)
    - normalizedScore (Float) - 정규화된 점수 (0-100)
    - testDate (DateTime) - 시험일
    - academicYear (Int) - 학년도
    - semester (Int) - 학기 (1, 2)
    - notes (String?) - 메모
    - createdAt/updatedAt
    - @@index([studentId, subject, testDate]) - 성적 추적용 인덱스
    
    **CounselingSession 모델:**
    - id (UUID, PK)
    - studentId (UUID, FK)
    - teacherId (String, FK)
    - sessionDate (DateTime) - 상담일
    - duration (Int) - 분 단위 상담 시간
    - type (Enum: ACADEMIC, CAREER, PSYCHOLOGICAL, BEHAVIORAL) - 상담 유형
    - summary (String) - 상담 요약
    - followUpRequired (Boolean) - 후속 조치 필요 여부
    - followUpDate (DateTime?) - 예정된 후속 상담일
    - satisfactionScore (Int?) - 학생 만족도 (1-5)
    - createdAt/updatedAt
    
    **StudentSatisfaction 모델:**
    - id (UUID, PK)
    - studentId (UUID, FK)
    - teacherId (String, FK)
    - surveyDate (DateTime) - 설문일
    - overallSatisfaction (Int) - 전반적 만족도 (1-10)
    - teachingQuality (Int) - 교육 품질 (1-10)
    - communication (Int) - 소통 (1-10)
    - supportLevel (Int) - 지원 수준 (1-10)
    - feedback (String?) - 자유 의견
    - createdAt/updatedAt
    
    **ControlVariables (학생별 통제 변수):**
    - Student 모델에 추가 필드:
      - initialGradeLevel (String?) - 입학 시 성적 수준 (HIGH/MID/LOW)
      - attendanceRate (Float?) - 출석률 (0-100)
      - priorAcademicPerformance (Json?) - 이전 성적 데이터
    
    **주의:**
    - 기존 모델과의 외래 키 관계를 올바르게 설정
    - ON DELETE CASCADE 또는 SET NULL 정책 결정 (student 삭제 시)
    - RLS 정책을 위한 teamId 필드 자동 설정 (Prisma Extension에서 처리)
    
    마이그레이션 실행: npx prisma migrate dev --name add_performance_models
    실패 시: npx prisma db push (shadow DB 동기화 이슈 대응)
