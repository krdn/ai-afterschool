  <name>Task 1: Student 모델에 teamId 추가</name>
  <files>prisma/schema.prisma</files>
  <action>
    prisma/schema.prisma 파일의 Student 모델을 수정하여 teamId 필드와 Team 관계를 추가:

    기존 Student 모델:
    ```prisma
    model Student {
      id                             String           @id @default(cuid())
      name                           String
      // ... 다른 필드들
      teacherId                      String
      createdAt                      DateTime         @default(now())
      // ...
      teacher                        Teacher          @relation(fields: [teacherId], references: [id])
      images                         StudentImage[]
    }
    ```

    변경 후 (teacherId 필드 뒤에 추가):
    ```prisma
    model Student {
      id                             String           @id @default(cuid())
      name                           String
      // ... 다른 필드들
      teacherId                      String
      teamId                         String?
      createdAt                      DateTime         @default(now())
      // ...
      teacher                        Teacher          @relation(fields: [teacherId], references: [id])
      team                           Team?            @relation(fields: [teamId], references: [id])
      images                         StudentImage[]
    }
    ```

    주요 변경 사항:
    1. teamId: String? 필드 추가 (nullable, 기존 데이터 호환)
    2. team: Team? 관계 추가 (단일 팀 소속)
    3. Team 모델 쪽에는 students 관계가 이미 존재하므로 추가 수정 불필요

    제약조건:
    - teamId는 NULL 허용 (기존 학생 데이터는 팀이 없음)
    - 외래 키 제약조건은 Prisma가 자동 생성
    - ON DELETE SET NULL: 팀 삭제 시 학생의 teamId를 NULL로 설정

    주의: Team 모델의 students 관계는 이미 정의되어 있으므로 Student 모델만 수정
  </action>
  <verify>prisma validate 명령 실행하여 스키마 유효성 확인</verify>

<task type="auto">
  <name>Task 2: 마이그레이션 생성 (수동 편집)</name>
  <files>prisma/migrations</files>
  <action>
    다음 단계를 따라 마이그레이션을 생성하고 수동으로 편집하여 NOT VALID 제약조건 추가:

    1. 마이그레이션 생성:
       ```bash
       npx prisma migrate dev --name add_student_team_id --create-only
       ```

    2. 생성된 마이그레이션 파일을 열어서 SQL을 확인하고 수정:
       파일 경로: prisma/migrations/[timestamp]_add_student_team_id/migration.sql

    원래 자동 생성된 SQL은 다음과 같을 것임:
    ```sql
    -- AlterTable
    ALTER TABLE "Student" ADD COLUMN "teamId" TEXT;
    ```

    3. 이를 무중단 마이그레이션으로 수정:

    ```sql
    -- AlterTable
    ALTER TABLE "Student" ADD COLUMN "teamId" TEXT;

    -- CreateIndex (NOT VALID for non-blocking)
    CREATE INDEX CONCURRENTLY "Student_teamId_idx" ON "Student"("teamId");

    -- AddForeignKeyConstraint (deferred to avoid lock)
    ALTER TABLE "Student" ADD CONSTRAINT "Student_teamId_fkey" FOREIGN KEY ("teamId") REFERENCES "Team"("id") ON DELETE SET NULL ON UPDATE CASCADE;
    ```

    주의: Prisma의 마이그레이션 시스템은 CONCURRENTLY를 직접 지원하지 않으므로 수동으로 수정 필요

    4. 수정 후 마이그레이션 적용:
       ```bash
       npx prisma migrate dev
       ```

    5. Prisma Client 재생성:
       ```bash
       npx prisma generate
       ```

    중요 사항:
    - NOT NULL 제약조건은 아직 추가하지 않음 (기존 데이터가 NULL이므로)
    - 외래 키 제약조건은 즉시 적용 (teamId가 NULL이어도 문제 없음)
    - 인덱스는 CONCURRENTLY로 생성 (LOCK 방지)
  </action>
  <verify>prisma migrate status 명령으로 마이그레이션 상태 확인</verify>
  <done>마이그레이션이 성공적으로 생성되고 적용되며 Student.teamId 컬럼 존재</done>
</task>
