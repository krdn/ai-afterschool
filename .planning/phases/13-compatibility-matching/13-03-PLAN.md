<task type="auto">
  <name>Task 2: 자동 배정 탐욕 알고리즘 구현</name>
  <files>src/lib/analysis/auto-assignment.ts</files>
  <action>
  1. src/lib/analysis/auto-assignment.ts 생성

  2. 타입 정의:
     - AssignmentInput: students[], teachers[], compatibilities[]
     - TeacherLoad: { teacherId, currentCount, minTarget, maxTarget }
     - AssignmentOutput: { assignments: Map<studentId, teacherId>, metrics: {...} }

  3. generateAutoAssignments 함수 구현 (탐욕 알고리즘):
     a) 선생님 부하 계산:
        - currentCount: 현재 학생 수
        - minTarget: 10, maxTarget: 30
        - loadFactor: currentCount / maxTarget (0-1)

     b) 모든 선생님-학생 조합에 대해 점수 계산:
        - compatibilityScore: getCompatibilityResult 또는 calculateCompatibilityScore
        - loadPenalty: loadFactor * 100 * 0.15 (부하 가중치)
        - adjustedScore: compatibilityScore - loadPenalty

     c) 탐욕 배정 (Greedy):
        - 학생 순회 (무작위 셔플 또는 정렬 기준)
        - 각 학생에 대해:
          i. availableTeachers = maxTarget 미달인 선생님 필터
          ii. availableTeachers 중 adjustedScore 최대인 선생님 선택
          iii. 해당 선생님 currentCount 증가
          iv. assignments[studentId] = selectedTeacherId

     d) 메트릭 계산:
        - avgCompatibility: 모든 배정의 평균 궁합 점수
        - loadBalanceScore: 표준 편차 기준 (낮을수록 좋음)
        - minStudentCount, maxStudentCount
        - teacherUtilization: 각 선생님의 학생 수 배열

     e) 결과 반환:
        {
          assignments: Record<string, string>, // studentId -> teacherId
          metrics: {
            avgCompatibility: number,
            loadBalanceScore: number,
            minStudentCount: number,
            maxStudentCount: number,
            teacherUtilization: Array<{teacherId, name, count}>
          }
        }

  4. 제약 조건 검증:
     - 모든 학생이 배정되었는지 확인
     - 어떤 선생님도 maxTarget 초과하지 않았는지 확인
  </action>
  <verify>
  grep -c "generateAutoAssignments\|Greedy\|loadFactor" src/lib/analysis/auto-assignment.ts
  </verify>
  <done>
  generateAutoAssignments 함수가 탐욕 알고리즘으로 배정을 생성하고 메트릭을 반환한다
  </done>
</task>
