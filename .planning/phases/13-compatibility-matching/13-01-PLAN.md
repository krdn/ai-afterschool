     - CompatibilityOutput: { overallScore: number, componentScores: ComponentScores, reasoning: string }

  3. calculateCompatibilityScore 함수 구현 (순수 함수):
     a) MBTI 호환도 (25%):
        - 차원별(E/I, S/N, T/F, J/P) 퍼센트 차이 계산
        - 같은 성향일수록 높은 점수 (선생님 E 60% vs 학생 E 60% = 100점)
        - 차이가 30% 이하면 80점, 30-60%면 60점, 60% 이상이면 40점
        - 4개 차원 평균

     b) 학습 스타일 호환도 (25%):
        - MBTI 퍼센트에서 유도 (S=Visual, N=Intuitive, T=Analytical, F=Social, J=Sequential, P=Global)
        - 같은 학습 스타일일수록 높은 점수 (가이드용, 과학적 근거 없음 주석 추가)
        - 6개 스타일 중 매칭 개수 / 6 * 100

     c) 사주 호환도 (20%):
        - 오행 균형 상호 보완 점수
        - 학생의 약한 오행을 선생님이 보유하면 +20점
        - 선생님의 강한 오행과 학생의 강한 오행이 같으면 -10점 (균형 저해)
        - 기본 60점에서 가감

     d) 성명학 호환도 (15%):
        - 수리 궁합 계산 (원형 수리 매칭)
        - 수리 합이 3, 5, 7, 8, 9면 100점 (길수)
        - 1, 2, 4, 6이면 70점
        - 나머지는 50점

     e) 부하 분산 점수 (15%):
        - 선생님 현재 학생 수 기반
        - 0-10명: 100점
        - 11-20명: 80점
        - 21-30명: 60점
        - 31명 이상: 40점

     f) 가중 평균 계산:
       overallScore = mbti*0.25 + learningStyle*0.25 + saju*0.20 + name*0.15 + loadBalance*0.15

     g) 추천 이유 생성 (reasoning):
        - "MBTI: [E/I 차이 설명], 학습 스타일: [매칭된 스타일], 사주: [보완 관계], 부하: [현재 학생 수명]"

  4. 학습 스타일 미신 경고 주석 추가:
