# Phase 12: Teacher Analysis & Team Data Access - Context

**Gathered:** 2026-01-30
**Status:** Ready for planning

<domain>
## Phase Boundary

선생님 성향 분석 시스템 구축. MBTI, 사주, 성명학, 관상, 손금 분석을 제공하고 기존 학생 분석 모듈을 재사용하여 통합 프로필 페이지를 구축합니다. 새로운 분석 알고리즘 개발이 아니라 **기존 모듈 재사용**이 핵심입니다.

</domain>

<decisions>
## Implementation Decisions

### 데이터 구조
- **별도 TeacherAnalysis 테이블**: Student 테이블을 공유하지 않고 독립적인 테이블 생성
- **모델 분리 + 로직 공유**: Prisma 모델은 별도로 만들되, 계산 로직(libraries/)은 Student와 공유
- **동일한 이미지 구조**: 관상/손금 분석 결과 이미지 저장 방식은 Student와 완전히 동일한 JSON 구조
- **분석 상태 추적**: TeacherAnalysis 테이블에 분석 완료 여부, 최종 수정일, 다음 분석 권장일 메타데이터 추가

### 분석 진행 방식
- **선생님 직접 입력**: 본인 계정으로 로그인 후 자신의 분석 직접 입력/관리 (관리자 대리 입력 불가)
- **전체 필수**: 모든 분석(MBTI, 사주, 성명학, 관상, 손금) 필수 완료
- **학생과 동일한 순서**: MBTI → 사주 → 성명학 → 관상/손금 순서로 진행
- **프로필 페이지에서 진행**: 선생님 계정에서 본인 프로필 페이지(/teachers/[id])로 바로 이동하여 입력
- **즉시 계산**: 각 분석 저장 후 바로 계산 (학생과 동일한 방식)

### 프로필 UI
- **학생과 동일한 디자인**: 선생님 프로필 페이지 레이아웃/스타일은 Student와 동일하게 유지
- **단일 페이지 (스크롤)**: 모든 분석 결과를 한 페이지에 스크롤하여 표시
- **헤더에 프로필 카드**: 상단에 선생님 기본 정보(이름, 사진, 소속 팀, 역할) 카드 표시
- **학생과 동일한 시각화**: MBTI 차트, 사주 오행 등 시각화 스타일은 Recharts, 컬러 팔레트 동일하게 적용

### 팀 데이터 접근
- **분석 상태 표시**: 선생님 목록 페이지에 각 선생님의 분석 완료 상태 표시
- **관리자 전용 조회**: 팀장/원장만 다른 선생님 분석 결과 조회 가능
- **기본 정보 + 상태 공개**: 비관리자에게 다른 선생님의 이름, 이메일, 팀, 역할 + 분석 완료 상태만 공개
- **목록 페이지 필터**: 선생님 목록 페이지에서 검색/필터 기능으로 팀 내 분석 현황 확인

### Claude's Discretion
- TeacherAnalysis 테이블의 정확한 필드 구조 (재사용 범위 구체화)
- 분석 상태 표시의 UI 구체적 구현 (진행률 바, 배지, 텍스트 등)
- 검색/필터 기능의 세부 옵션 (분석 완료/미완료, 팀별, 역할별 등)

</decisions>
