<task type="checkpoint:human-verify" gate="blocking">
  <what-built>Complete counseling section integration on student detail page with:
- CounselingSection component displaying upcoming reservation alert and history list
- CounselingSessionModal for session details
- Data fetching for sessions and upcoming reservations
- Click handlers for opening modal from session cards</what-built>
  <how-to-verify>
    1. Start dev server if not running:
       ```bash
       npm run dev
       ```

    2. Navigate to a student detail page:
       - Go to http://localhost:3000/students
       - Click on any student to view their detail page

    3. Verify counseling section exists:
       - Scroll to bottom of page
       - Look for "상담 이력" section header (h2)
       - Section should be below "상담 보고서" section

    4. Verify upcoming reservation display (if exists):
       - Look for blue Alert box with Calendar icon
       - Check for "다음 상담 예약" title
       - Verify date format: "M월 d일 E요일 HH:mm" (e.g., "2월 5일 수요일 14:00")
       - Check parent name and relation display
       - Check topic display

    5. Verify counseling history display:
       - Check for month-grouped sessions (e.g., "2025년 2월")
       - Verify session cards show: relative time, type badge, teacher name, duration, summary
       - Verify follow-up indicator (if applicable)
       - Verify satisfaction stars (if applicable)

    6. Test session detail modal:
       - Click on any session card
       - Modal should open with title "상담 상세"
       - Verify all fields display: type, duration, teacher, summary, follow-up, satisfaction
       - Click outside modal or press Escape to close
       - Modal should close

    7. Test empty state (if no sessions):
       - Find a student with no counseling history
       - Verify "아직 상담 기록이 없습니다" message displays

    8. Test security (teacherId filtering):
       - Note current teacher ID from session
       - Try accessing student belonging to different teacher (if possible)
       - Should not see counseling data for other teacher's students

    9. Visual inspection:
       - Check spacing and layout consistency with other sections
       - Verify Alert styling matches overall design
       - Check hover effects on session cards
       - Verify modal styling and centering
  </how-to-verify>
  <resume-signal>Type "approved" if all checks pass, or describe any issues found</resume-signal>
