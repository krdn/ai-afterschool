# Phase 20: Student Page Integration - Context

**Gathered:** 2026-02-04
**Status:** Ready for planning

## Phase Boundary

학생 상세 페이지에 상담 기능을 통합하여 기존 UI를 유지하면서 상담 탭, 이력 조회, 다음 예약 표시를 추가합니다. 기존 학생 상세 페이지의 UI/UX 패턴을 따르며 점진적 통합으로 회귀 위험을 최소화합니다.

## Implementation Decisions

### 탭 구조 및 배치
- **탭 위치**: 기존 탭 뒤에 마지막에 추가
- **탭 라벨**: "상담 이력"
- **탭 전환 방식**: 기존 학생 페이지의 탭 전환 패턴을 확인하고 동일하게 적용 (Claude가 코드 확인 후 결정)
- **탭 내용 구성**: 기존 예약 관리 페이지 패턴을 참고하여 결정 (Claude가 결정)

### 상담 이력 표시
- **정렬 순서**: 일반적인 패턴 사용 (Claude가 결정)
- **카드 정보**: 기존 CounselingHistoryList 컴포넌트 패턴 사용 (Claude가 결정)
- **필터링**: 기존 패턴 고려 (Claude가 결정)
- **클릭 동작**: 상세 모달로 상담 내용 표시

### 다음 예약 표시
- **표시 위치**: UI 흐름을 고려하여 결정 (페이지 상단 또는 탭 내 상단)
- **표시 스타일**: shadcn/ui Alert 패턴 사용
- **표시 정보**: 기존 예약 카드 패턴 사용 (Claude가 결정)
- **예약 없을 때**: 기존 빈 상태 패턴 사용 (Claude가 결정)

### 빈 상태 및 로딩
- **빈 상태**: 기존 빈 상태 패턴 사용
- **로딩 상태**: 기존 로딩 패턴 사용
- **에러 상태**: 기존 에러 처리 패턴 사용

### Claude's Discretion
다음 항목은 Claude가 기존 코드 패턴을 확인 후 결정합니다:
- 탭 전환 방식 (URL 쿼리 파라미터 vs 클라이언트 상태 vs 라우팅)
- 상담 이력 정렬 순서
- 이력 카드에 표시할 정보 구성
- 필터링 옵션
- 다음 예약 표시 위치와 스타일
- 예약 없을 때의 처리 방식
- 빈 상태/로딩/에러 UI 구체적 구현

## Specific Ideas

- 기존 학생 상세 페이지 UI를 파괴하지 않고 점진적 통합
- 회귀 테스트로 기존 기능 보호
