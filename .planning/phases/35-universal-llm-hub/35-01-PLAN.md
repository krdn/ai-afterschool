<task type="auto">
  <name>Task 1: 새로운 Prisma 스키마 정의</name>
  <files>prisma/schema.prisma</files>
  <action>
    기존 LLMConfig, LLMFeatureConfig, LLMBudget 테이블 아래에 새로운 테이블 추가:

    1. Provider 테이블:
       - id: String @id @default(cuid())
       - name: String (표시명)
       - providerType: String ('openai', 'anthropic', 'ollama', 'custom', etc.)
       - baseUrl: String? (API 엔드포인트)
       - apiKeyEncrypted: String? (암호화된 API 키)
       - authType: String ('api_key', 'bearer', 'custom_header')
       - customAuthHeader: String? (커스텀 인증 시 헤더명)
       - capabilities: String[] (JSON 배열: 'vision', 'function_calling', 'json_mode', 'streaming')
       - costTier: String ('free', 'low', 'medium', 'high')
       - qualityTier: String ('fast', 'balanced', 'premium')
       - isEnabled: Boolean @default(false)
       - isValidated: Boolean @default(false)
       - validatedAt: DateTime?
       - createdAt, updatedAt

    2. Model 테이블:
       - id: String @id @default(cuid())
       - providerId: String (FK)
       - modelId: String (API용 실제 모델 ID)
       - displayName: String
       - contextWindow: Int?
       - supportsVision: Boolean @default(false)
       - supportsTools: Boolean @default(false)
       - defaultParams: Json? (temperature, maxTokens 등)
       - isDefault: Boolean @default(false)
       - createdAt, updatedAt
       - @@unique([providerId, modelId])

    3. FeatureMapping 테이블:
       - id: String @id @default(cuid())
       - featureType: String (기능 유형)
       - matchMode: String ('auto_tag', 'specific_model')
       - requiredTags: String[] (필요 태그 목록)
       - excludedTags: String[] (제외 태그 목록)
       - specificModelId: String? (직접 지정 시)
       - priority: Int @default(1)
       - fallbackMode: String ('next_priority', 'any_available', 'fail')
       - createdAt, updatedAt

    4. ProviderTemplate 테이블 (미리 정의된 템플릿):
       - id: String @id @default(cuid())
       - templateId: String @unique (예: 'openai', 'anthropic')
       - name: String
       - providerType: String
       - defaultBaseUrl: String?
       - defaultCapabilities: String[]
       - defaultAuthType: String
       - description: String
       - helpUrl: String?
       - isPopular: Boolean @default(false)
       - sortOrder: Int @default(0)

    참고: 기존 테이블(LLMConfig, LLMFeatureConfig, LLMBudget)은 유지하여 하위호환성 보장
  </action>
  <verify>grep -A 5 "model Provider" prisma/schema.prisma && grep -A 5 "model Model" prisma/schema.prisma && grep -A 5 "model FeatureMapping" prisma/schema.prisma</verify>
  <done>새로운 Provider, Model, FeatureMapping, ProviderTemplate 모델이 schema.prisma에 정의됨</done>
</task>
