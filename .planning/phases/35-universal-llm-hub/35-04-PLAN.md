  <name>Task 1: FeatureResolver 구현</name>
  <files>src/lib/ai/feature-resolver.ts</files>
  <action>
    src/lib/ai/feature-resolver.ts 생성:

    FeatureResolver 클래스:
    - constructor(private db: PrismaClient)

    메서드:
    1. async resolve(featureType: string, requirements?: ResolutionRequirements): Promise<ResolutionResult>
       - 주어진 기능에 가장 적합한 모델을 찾음
       - requirements: { needsVision?: boolean; preferredCost?: CostTier; }
       - 반환: { provider: ProviderConfig; model: ModelConfig; priority: number }

    2. async resolveWithFallback(featureType: string, requirements?: ResolutionRequirements): Promise<ResolutionResult[]>
       - 우선순위 순으로 정렬된 후보 목록 반환
       - 폴 백 체인 구성에 사용

    3. private async resolveByTags(featureType: string, requirements: ResolutionRequirements): Promise<ResolutionResult[]>
       - FeatureMapping에서 auto_tag 규칙 조회
       - requiredTags와 excludedTags로 필터링
       - 예산, 성능 요구사항으로 추가 필터링
       - 우선순위로 정렬

    4. private async resolveBySpecificModel(featureType: string): Promise<ResolutionResult | null>
       - FeatureMapping에서 specific_model 규칙 조회
       - 지정된 모델 직접 반환

    5. async getMappings(featureType?: string): Promise<FeatureMappingWithDetails[]>
       - 매핑 규칙 조회 (특정 기능 또는 전체)

    6. async createOrUpdateMapping(input: FeatureMappingInput): Promise<FeatureMappingConfig>
       - 매핑 규칙 생성/수정

    7. async deleteMapping(id: string): Promise<void>
       - 매핑 규칙 삭제

    ResolutionRequirements 인터페이스:
    - needsVision?: boolean - Vision 필요 여부
    - needsTools?: boolean - Function calling 필요 여부
    - preferredCost?: CostTier - 선호 비용 등급
    - preferredQuality?: QualityTier - 선호 품질 등급
    - minContextWindow?: number - 최소 컨텍스트 윈도우
  </action>
