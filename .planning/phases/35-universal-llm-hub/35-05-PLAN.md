    src/lib/ai/universal-router.ts 생성:

    기존 router.ts의 인터페이스를 유지하면서 남부 구현을 Universal LLM Hub로 교체

    인터페이스 (기존과 동일):
    - GenerateOptions
    - VisionGenerateOptions
    - GenerateResult
    - FailoverError (재사용)

    핵심 함수:
    1. generateWithProvider(options: GenerateOptions): Promise<GenerateResult>
       - FeatureResolver.resolve()로 모델 선택
       - ProviderRegistry.getAdapter()로 어댑터 획득
       - 어댑터.generate() 호출
       - 실패 시 폴 백 체인 순회
       - 사용량 추적 (trackUsage)

    2. streamWithProvider(options: GenerateOptions): Promise<StreamResult>
       - 비슷한 패턴으로 스트리밍 구현

    3. generateWithVision(options: VisionGenerateOptions): Promise<GenerateResult>
       - needsVision: true 요구사항 전달
       - Vision 지원 모델만 필터링

    4. generateWithSpecificProvider(provider: string, options): Promise<GenerateResult>
       - 특정 제공자 강제 사용

    5. generateVisionWithSpecificProvider(provider: string, options): Promise<GenerateResult>
       - 특정 제공자 Vision 강제 사용

    남부 구현:
    - FeatureResolver를 통해 기능->모델 해결
    - ProviderRegistry를 통해 제공자 관리
    - Adapter 패턴으로 실제 API 호출
    - 기존 usage-tracker, failover, smart-routing 재사용
  </action>
