# Phase 28-01: Test Environment Setup & Initial Run

**Wave:** 1
**Autonomous:** Yes
**Depends on:** None

---

## Tasks

### Task 1: Verify Test Environment Readiness

**Action:** Verify all prerequisites for E2E test execution are met.

**Steps:**
1. Check Playwright installation: `npx playwright --version`
2. Verify browsers installed: `npx playwright install chromium`
3. Check test config exists: `cat playwright.config.ts`
4. Ensure dev server can start: `npm run dev &` (check for errors)
5. Verify database connectivity: Check DATABASE_URL in .env

**Verification:**
- Playwright v1.58.1+ installed
- Chromium browser available
- Dev server starts without errors
- Database connection successful

---

### Task 2: Run Initial E2E Test Suite

**Action:** Execute full E2E test suite and capture all failures.

**Steps:**
1. Start dev server in background: `npm run dev &`
2. Wait for server ready (http://localhost:3000 responds)
3. Run full test suite: `npm run test:e2e -- --reporter=json > test-results.json`
4. Parse results and categorize failures
5. Create failure summary document

**Commands:**
```bash
# Background dev server
npm run dev &
DEV_PID=$!

# Wait for ready
sleep 10

# Run tests
npm run test:e2e -- --reporter=json > test-results.json 2>&1

# Kill dev server
kill $DEV_PID
```

**Output:** `.planning/phases/28-integration-verification-&-test-alignment/TEST-FAILURES.md`

---

### Task 3: Categorize Test Failures

**Action:** Analyze test results and categorize by root cause.

**Categories:**
1. **Selector Issues** - Missing or incorrect data-testid, CSS selectors
2. **Auth/Session Issues** - Login failures, redirect loops
3. **Timing Issues** - Elements not ready, race conditions
4. **Data Issues** - Missing test fixtures, database state
5. **Logic Issues** - Actual bugs in implementation

**Template:**
```markdown
# Test Failure Analysis

## Summary
Total: 74 tests
Passed: X
Failed: Y

## By Category
| Category | Count | Tests |
|----------|-------|-------|
| Selector | ? | auth.spec.ts::AUTH-01, ... |
| Auth | ? | ... |
| Timing | ? | ... |
```

**Verification:**
- All 74 tests categorized
- Failure patterns identified
- Priority fix order determined
