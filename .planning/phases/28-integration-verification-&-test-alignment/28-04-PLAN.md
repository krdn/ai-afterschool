### Task 4: Create Test Maintenance Guide

**Action:** Document how to keep tests passing going forward.

**Create:** `docs/qa/TEST-MAINTENANCE.md`

```markdown
# E2E Test Maintenance Guide

## Running Tests

### Full Suite
```bash
npm run dev &
npm run test:e2e
```

### Single Spec File
```bash
npm run test:e2e -- auth.spec.ts
```

### Debug Mode
```bash
npm run test:e2e -- --debug
```

## Adding New Tests

1. Create new spec file in `tests/e2e/`
2. Use `data-testid` selectors (not text/CSS)
3. Add test ID to component when creating feature
4. Update `tests/utils/selectors.ts` if reusing selectors

## When Tests Fail

1. Run with `--reporter=html` for detailed output
2. Check `playwright-report/index.html`
3. Categorize failure: selector vs logic vs environment
4. Fix and re-run specific test file

## Selector Conventions

- Use `data-testid` for component identification
- Pattern: `[component]-[element]` (e.g., `student-delete-button`)
- Use `data-tab` for tab buttons
- Avoid text-based selectors (fragile)
```
