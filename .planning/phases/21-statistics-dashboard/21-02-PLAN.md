   - includeCompleted?: boolean
   - teacherId?: string

  <name>Task 2: 후속 조치 Server Actions 구현</name>
  <files>src/lib/actions/follow-up.ts</files>
  <action>
CounselingSession의 followUpRequired, followUpDate 필드를 활용한 Server Actions 구현:

1. getFollowUpsAction(filter: FollowUpFilter)
   - RBAC: verifySession() + getRBACPrisma(session)
   - 쿼리 조건:
     - followUpRequired: true
     - scope 'today': followUpDate가 오늘 (startOfDay ~ endOfDay)
     - scope 'week': followUpDate가 이번 주 내 (startOfWeek ~ endOfWeek)
     - includeCompleted가 false면 satisfactionScore가 null인 것만 (미완료)
   - 지연 상태 계산: followUpDate < today && 미완료 -> 'overdue'
   - 정렬: followUpDate ASC (마감일 임박순)
   - 반환: { success, data: FollowUpItem[] }

2. completeFollowUpAction(input: CompleteFollowUpInput)
   - RBAC: 해당 세션의 teacherId와 현재 사용자 일치 확인
   - 업데이트:
     - satisfactionScore를 임시값(1)으로 설정하여 완료 표시 (기존 스키마 활용)
     - 또는 새 필드 추가가 필요하면 summary에 완료 메모 추가
   - Note: 기존 스키마에 followUpCompleted 필드가 없으므로, satisfactionScore !== null을 완료 기준으로 사용
   - revalidatePath('/dashboard/statistics')
   - 반환: { success, data: { id, completedAt } }

3. getOverdueCountAction()
   - followUpRequired: true
   - followUpDate < today
   - satisfactionScore: null (미완료)
   - 반환: { success, data: { count } }

날짜 계산에 date-fns 사용:
- startOfDay, endOfDay
- startOfWeek, endOfWeek (locale: ko)
- isBefore (지연 판단)
  </action>
