src/lib/actions/analytics.ts의 getCounselingStats 패턴을 참조하여 확장된 통계 Server Actions를 구현합니다:

1. getTeacherMonthlyStatsAction(params: { dateFrom?: string, dateTo?: string })
   - RBAC: verifySession() + getRBACPrisma(session)
   - Prisma groupBy로 teacherId, year, month별 집계
   - 선생님 이름 조인하여 반환
   - 반환: { success, data: TeacherMonthlyStats[] }

2. getStudentCumulativeStatsAction(params: { teacherId?: string })
   - RBAC: TEACHER는 자신 담당 학생만
   - 학생별 CounselingSession 카운트 집계
   - 마지막 상담 날짜 포함
   - 반환: { success, data: StudentCumulativeStats[] }

3. getCounselingTypeDistributionAction(params: { dateFrom?: string, dateTo?: string, teacherId?: string })
   - RBAC 적용
   - CounselingType별 groupBy
   - 비율(percentage) 계산 포함
   - 반환: { success, data: TypeDistribution[] }

4. getMonthlyTrendAction(params: { months?: number, teacherId?: string })
   - 기본값 months: 6
   - date-fns의 startOfMonth, subMonths로 날짜 범위 계산
   - 월별 상담 건수 집계
   - 반환: { success, data: MonthlyTrend[] }

5. getDateRangeFromPreset(preset: DatePreset): DateRange
   - 유틸리티 함수 (export)
   - date-fns 사용: startOfMonth, endOfMonth, subMonths

공통 에러 처리 패턴:
- try/catch로 감싸기
- console.error로 로깅
- { success: false, error: string } 반환
