`src/lib/github/services.ts`를 생성한다. GitHub API 호출을 캡슐화하는 서비스 계층이다.

**구현할 함수 3개:**

**1. `createGitHubIssue()`:**
- octokit.rest.issues.create 호출
- title, body, labels 파라미터 받기
- body에 이슈 메타데이터(카테고리, 우선순위, 생성자) 포함
- 응답에서 issue number, html_url 추출
- 응답 헤더에서 rate limit 체크 (checkRateLimitFromHeaders 호출)
- 실패 시 null 반환 (throw하지 않음 — DB 이슈는 GitHub 실패와 무관하게 생성되어야 함)
- 에러 발생 시 logSystemAction으로 에러 기록

```typescript
export async function createGitHubIssue(params: {
  title: string
  body: string
  labels: string[]
}): Promise<{ number: number; htmlUrl: string } | null>
```

**2. `ensureLabel()`:**
- 라벨이 저장소에 존재하는지 확인하고, 없으면 생성
- LABEL_COLORS, LABEL_DESCRIPTIONS에서 색상/설명 가져오기
- 이미 존재하면 아무것도 하지 않음 (idempotent)
- 실패 시 경고 로그만 기록 (critical이 아님)

```typescript
export async function ensureLabel(labelName: string): Promise<boolean>
```

**3. `createIssueBranch()`:**
- main 브랜치의 최신 커밋 SHA 가져오기 (octokit.rest.git.getRef)
- 새 브랜치 생성 (octokit.rest.git.createRef)
- branchName은 generateBranchName() 유틸리티 사용
- 이미 존재하는 브랜치면 에러를 무시하고 기존 브랜치명 반환
- 실패 시 null 반환 + 에러 로그

```typescript
export async function createIssueBranch(params: {
  issueNumber: number
  title: string
  category: IssueCategory
}): Promise<string | null>
```

**모든 함수 공통 패턴:**
- `import 'server-only'` 최상단
- try/catch로 GitHub API 에러 격리
- logSystemAction으로 에러/경고 기록
- getOctokit(), getRepoConfig()에서 설정 가져오기
- isGitHubConfigured() 체크 → false면 null 반환 (GitHub 미설정 환경 대응)
- 응답 헤더에서 rate limit 확인

```markdown
**이슈 정보:**

- 앱에서 생성됨
```

**1. `src/lib/validations/issues.ts` 생성:**

이슈 생성 폼의 입력 검증 Zod 스키마를 정의한다.

```typescript
import { z } from "zod"

export const IssueSchema = z.object({
  title: z.string()
    .min(3, "제목은 3자 이상이어야 해요")
    .max(200, "제목은 200자 이내여야 해요"),
  description: z.string()
    .max(5000, "설명은 5000자 이내여야 해요")
    .optional()
    .or(z.literal('')),
  category: z.enum([
    'BUG', 'FEATURE', 'IMPROVEMENT', 'UI_UX',
    'DOCUMENTATION', 'PERFORMANCE', 'SECURITY'
  ], { message: "카테고리를 선택해 주세요" }),
  priority: z.enum(['LOW', 'MEDIUM', 'HIGH', 'URGENT'])
    .default('MEDIUM'),
})

export type IssueFormState = {
  errors?: {
    title?: string[]
    description?: string[]
    category?: string[]
    priority?: string[]
    _form?: string[]
  }
  message?: string
  success?: boolean
  issueId?: string
}
```

```typescript
"use server"

import { verifySession, logAuditAction } from "@/lib/dal"
import { db } from "@/lib/db"
import { IssueSchema, type IssueFormState } from "@/lib/validations/issues"
import { createGitHubIssue, ensureLabel, createIssueBranch } from "@/lib/github/services"
import { CATEGORY_LABEL_MAP } from "@/lib/github/constants"
import { isGitHubConfigured } from "@/lib/github/client"
import type { IssueCategory, IssuePriority } from "@prisma/client"

export async function createIssue(
  prevState: IssueFormState,
  formData: FormData
): Promise<IssueFormState>
```

**핵심 흐름:**

Step 1: 인증 + DIRECTOR 권한 체크
```typescript
const session = await verifySession()
if (session.role !== 'DIRECTOR') {
  return { errors: { _form: ["이슈를 생성할 권한이 없어요"] } }
}
```

Step 2: Zod 검증
```typescript
const validatedFields = IssueSchema.safeParse({
  title: formData.get("title"),
  description: formData.get("description"),
  category: formData.get("category"),
  priority: formData.get("priority"),
})
```

Step 3: DB에 이슈 저장 (먼저 DB에 저장하여 GitHub 실패와 무관하게 이슈 보존)
```typescript
const issue = await db.issue.create({
  data: {
    title, description: description || null,
    category: category as IssueCategory,
    priority: (priority || 'MEDIUM') as IssuePriority,
    createdBy: session.userId,
  },
})
```

Step 4: GitHub 연동 (isGitHubConfigured() 체크 후)
- ensureLabel(카테고리 라벨)
- createGitHubIssue({ title, body, labels: [카테고리 라벨] })
- 성공 시: createIssueBranch({ issueNumber: githubIssue.number, title, category })
- DB 업데이트: githubIssueNumber, githubIssueUrl, githubBranchName

Step 5: IssueEvent 기록
- "created" 이벤트
- "labeled" 이벤트 (GitHub 라벨 태깅 성공 시)
- "branch_created" 이벤트 (브랜치 생성 성공 시)

Step 6: AuditLog 기록
```typescript
await logAuditAction({
  action: 'ISSUE_CREATED',
  entityType: 'Issue',
  entityId: issue.id,
  changes: { title, category, githubIssueNumber: githubIssue?.number },
})
```

Step 7: 결과 반환 (redirect 없음 — Phase 30에서 모달로 사용하므로)
```typescript
return { success: true, message: "이슈가 생성되었어요", issueId: issue.id }
```
