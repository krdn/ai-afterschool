minio:
  image: minio/minio:latest
  container_name: minio
  command: server /data --console-address ":9001"
  restart: unless-stopped
  ports:
    - "9000:9000"  # API
    - "9001:9001"  # Console (web UI)
  volumes:
    - minio_data:/data
  environment:
    MINIO_ROOT_USER: ${MINIO_ROOT_USER:-minioadmin}
    MINIO_ROOT_PASSWORD: ${MINIO_ROOT_PASSWORD:-minioadmin}
    MINIO_DEFAULT_BUCKETS: ${MINIO_BUCKET:-reports}
  healthcheck:
    test: ["CMD", "curl", "-f", "http://localhost:9000/minio/health/live"]
    interval: 30s
    timeout: 10s
    retries: 3
  networks:
    - backend

# Add volume
volumes:
  minio_data:
    driver: local
```

**Security notes:**
- Change default credentials in production
- MINIO_ROOT_USER and MINIO_ROOT_PASSWORD should be strong passwords
- Volume `minio_data` persists across container restarts

<action>Create MinIO bucket initialization script</action>
<file>scripts/setup-minio.ts</file>
<details>
Create `scripts/setup-minio.ts`:

```typescript
#!/usr/bin/env tsx
/**
 * Initialize MinIO bucket and policies
 * Run after: docker-compose -f docker-compose.prod.yml up -d
 */

import { S3Client, CreateBucketCommand, PutBucketPolicyCommand } from '@aws-sdk/client-s3'

const client = new S3Client({
  endpoint: process.env.MINIO_ENDPOINT || 'http://localhost:9000',
  region: process.env.MINIO_REGION || 'us-east-1',
  credentials: {
    accessKeyId: process.env.MINIO_ROOT_USER || 'minioadmin',
    secretAccessKey: process.env.MINIO_ROOT_PASSWORD || 'minioadmin',
  },
  forcePathStyle: true,
})

const BUCKET_NAME = process.env.MINIO_BUCKET || 'reports'

async function setupMinio() {
  console.log('🪣 Setting up MinIO...')

  // Create bucket
  try {
    await client.send(new CreateBucketCommand({ Bucket: BUCKET_NAME }))
    console.log(`✅ Bucket created: ${BUCKET_NAME}`)
  } catch (error: any) {
    if (error.Code !== 'BucketAlreadyExists' && error.Code !== 'BucketAlreadyOwnedByYou') {
      console.error('❌ Error creating bucket:', error)
      throw error
    }
    console.log(`ℹ️  Bucket already exists: ${BUCKET_NAME}`)
  }

  // Set bucket policy (public read for presigned URLs, private write)
  const policy = {
    Version: '2012-10-17',
    Statement: [
      {
        Effect: 'Allow',
        Principal: { AWS: ['*'] },
        Action: ['s3:GetObject'],
        Resource: [`arn:aws:s3:::${BUCKET_NAME}/*`],
      },
    ],
  }

  try {
    await client.send(new PutBucketPolicyCommand({
      Bucket: BUCKET_NAME,
      Policy: JSON.stringify(policy),
    }))
    console.log('✅ Bucket policy set: public read (presigned URLs required)')
  } catch (error) {
    console.error('❌ Error setting policy:', error)
    throw error
  }

  console.log('\n✨ MinIO setup complete!')
  console.log(`\n📊 Console: http://localhost:9001`)
  console.log(`🪣 Bucket: ${BUCKET_NAME}`)
}

setupMinio().catch(console.error)
```

Add to package.json scripts:
```json
"setup:minio": "tsx scripts/setup-minio.ts"
```

## MinIO S3-Compatible Storage

MinIO provides S3-compatible object storage for PDF reports.

### Access

**API (application use):** http://localhost:9000 (or minio:9000 in Docker network)
**Console (admin UI):** http://localhost:9001

Default credentials (CHANGE IN PRODUCTION):
- Username: minioadmin
- Password: minioadmin

### Operations

**Initialize bucket:**
```bash
npm run setup:minio
```

**Manual operations with mc (MinIO Client):**
```bash
# List files
mc ls local/reports

# Upload file
mc cp ./file.pdf local/reports/

# Download file
mc cp local/reports/file.pdf ./downloaded.pdf

# Remove file
mc rm local/reports/file.pdf
```

### Backup

MinIO data is stored in Docker volume `minio_data`. Backup:

```bash
# Create backup
docker run --rm -v minio_data:/data -v $(pwd):/backup \
  alpine tar czf /backup/minio-backup.tar.gz -C /data .

# Restore backup
docker run --rm -v minio_data:/data -v $(pwd):/backup \
  alpine tar xzf /backup/minio-backup.tar.gz -C /data
```

### Production Notes

- Change default credentials before production deployment
- Use strong passwords: `openssl rand -base64 32`
- Enable versioning for additional safety
- Consider lifecycle policies for old PDF cleanup
```

## Notes

- MinIO is compatible with AWS S3 API - same SDK works for both
- Using `forcePathStyle: true` is required for MinIO (vs path-based S3 URLs)
- Presigned URLs are the secure way to share files (public read policy allows presigned URLs)
- Default credentials are convenient for development but MUST be changed in production
- MinIO Console is exposed on port 9001 for management - consider restricting in production
