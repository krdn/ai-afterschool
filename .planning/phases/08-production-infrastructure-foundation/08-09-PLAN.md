S3 스토리지 사용 시 API 엔드포인트가 presigned URL 대신 PDF를 직접 반환하도록 수정합니다.

1. S3 storage 경로에서 JSON 반환({url: presignedUrl})을 제거
2. S3에서 PDF를 fetch하여 Buffer로 변환한 후 직접 반환
3. 로컬 스토리지와 S3 스토리지 모두에서 동일한 응답 형태(PDF 스트림) 사용

구현 패턴:
```typescript
// For S3 storage, fetch and proxy the PDF
if (process.env.PDF_STORAGE_TYPE === 's3') {
  const presignedUrl = await storage.getPresignedUrl(filename)
  const pdfResponse = await fetch(presignedUrl)
  const pdfBuffer = await pdfResponse.arrayBuffer()

  return new NextResponse(Buffer.from(pdfBuffer) as BodyInit, {
    headers: {
      'Content-Type': 'application/pdf',
      'Content-Disposition': `attachment; filename="report-${student.name}.pdf"`,
      'Content-Length': pdfBuffer.byteLength.toString(),
      'Cache-Control': 'public, max-age=3600',
    },
  })
}
```

Gap reason: "S3 storage 시 JSON {url: presignedUrl}을 반환하지만, 프론트엔드가 이를 올바르게 처리하지 않음" - API가 JSON 대신 PDF를 직접 반환하도록 하여 프론트엔드가 단일 URL로 다운로드 가능하도록 함

3. S3 스토리지(PDF_STORAGE_TYPE=s3) 설정 후 curl 테스트
  </verify>
