현재 상태 확인:

- health_check 함수는 실패 시 return 1을 반환
- deploy_new_version 함수는 health check 실패 시 return 1을 반환

GitHub Actions에서 이 종료 코드를 감지할 수 있도록:
1. deploy.sh의 마지막에 명시적인 종료 코드 반환 확인
2. 배포 실패 시 롤백이 호출되더라도 실패를 나타내는 종료 코드(1)를 반환하도록 수정

현재 롤백 로직(line 251-264):
```bash
if ! deploy_new_version; then
    log_error "Deployment failed at deploy stage"
    log_info "Rolling back to previous version..."
    # ... rollback logic ...
    exit 1  # 이미 존재함
fi
```

이미 exit 1이 있으므로 GitHub Actions가 실패를 감지할 수 있습니다. 추가 필요 사항:
- 롤백 후에도 exit 1을 반환하여 CI/CD가 실패로 인식하도록 확인
- rollback.sh 호출 후 명시적으로 exit 1 반환 확인

Gap reason: "배포 실패 시 수동으로 롤백 스크립트를 호출하지만, GitHub Actions에서 자동으로 처리되지 않음" - deploy.sh가 실패 시 적절한 종료 코드를 반환하도록 확인하여 CI/CD가 실패를 감지하고 롤백 단계로 진행할 수 있도록 함

<task type="auto">
  <name>Task 2: GitHub Actions 워크플로우에 자동 롤백 단계 추가</name>
  <files>.github/workflows/deploy.yml</files>
  <action>
GitHub Actions 워크플로우에 배포 실패 시 자동으로 롤백을 실행하는 단계를 추가합니다.

현재 워크플로우(line 15-24):
```yaml
- name: Deploy to server
  uses: appleboy/ssh-action@v1.0.0
  with:
    host: ${{ secrets.SERVER_HOST }}
    username: ${{ secrets.SERVER_USER }}
    key: ${{ secrets.SSH_PRIVATE_KEY }}
    script: |
      cd /path/to/ai-afterschool
      git pull origin main
      ./scripts/deploy.sh --force --tag=${{ github.sha }}
```

수정 사항:
1. deploy.sh 실행 후 실패 시 롤백을 실행하는 단계 추가
2. continue-on-error: true를 사용하여 롤백 단계가 항상 실행되도록 설정
3. 롤백 후 health check를 통해 복구 검증

수정된 워크플로우:
```yaml
- name: Deploy to server
  id: deploy
  uses: appleboy/ssh-action@v1.0.0
  continue-on-error: true
  with:
    host: ${{ secrets.SERVER_HOST }}
    username: ${{ secrets.SERVER_USER }}
    key: ${{ secrets.SSH_PRIVATE_KEY }}
    script: |
      cd /path/to/ai-afterschool
      git pull origin main
      ./scripts/deploy.sh --force --tag=${{ github.sha }}

- name: Rollback on failure
  if: failure() && steps.deploy.outcome == 'failure'
  uses: appleboy/ssh-action@v1.0.0
  with:
    host: ${{ secrets.SERVER_HOST }}
    username: ${{ secrets.SERVER_USER }}
    key: ${{ secrets.SSH_PRIVATE_KEY }}
    script: |
      cd /path/to/ai-afterschool
      ./scripts/rollback.sh --force

- name: Verify rollback health
  if: failure() && steps.deploy.outcome == 'failure'
  uses: appleboy/ssh-action@v1.0.0
  with:
    host: ${{ secrets.SERVER_HOST }}
    username: ${{ secrets.SERVER_USER }}
    key: ${{ secrets.SSH_PRIVATE_KEY }}
    script: |
      curl -f http://localhost:3000/api/health || exit 1

- name: Fail workflow
  if: failure() && steps.deploy.outcome == 'failure'
  run: exit 1
```

GitHub Actions 제약사항: SSH action에서 script가 실패해도 job이 실패로 표시되지 않을 수 있으므로, 명시적으로 실패를 감지하고 롤백을 실행합니다.

Gap reason: "배포 실패 시 롤백 단계가 없음" - GitHub Actions 워크플로우에 배포 실패 시 자동 롤백 단계를 추가하여 운영자 개입 없이 자동 복구
  </action>

2. 워크플로우 시각적 검토: deploy 단계 실패 시 rollback 단계가 실행되는지 확인

  </done>
