## Objective

Docker 기반 프로덕션 환경을 구성합니다. 멀티스테이지 빌드로 최적화된 이미지를 생성하고, PostgreSQL, MinIO, Next.js App, Caddy를 포함한 완전한 docker-compose 환경을 설정합니다.

## Tasks

<tasks>
<task>
<action>Create multi-stage production Dockerfile</action>
<file>Dockerfile.prod</file>
<details>
Create `Dockerfile.prod` with the following stages:

1. **base**: Node.js 20 Alpine base image
2. **deps**: Install production dependencies only
3. **builder**: Build Next.js application
4. **production**: Final runtime image with non-root user

Key requirements:
- Use `WORKDIR /app`
- Copy `package.json` and `package-lock.json` first for layer caching
- Run `npx prisma generate` in builder stage
- Copy `.next`, `prisma`, `public` to production stage
- Create `nextjs` user (UID 1001) and `nodejs` group (GID 1001)
- Set `NODE_ENV=production`
- Expose port 3000
- CMD: `node node_modules/.bin/next start`
</details>
</task>

<task>
<action>Create docker-compose.prod.yml with all services</action>
<file>docker-compose.prod.yml</file>
<details>
Create `docker-compose.prod.yml` with services:

**postgres:**
- Image: `postgres:16-alpine`
- Volume: `postgres_data:/var/lib/postgresql/data`
- Environment: `POSTGRES_DB`, `POSTGRES_USER`, `POSTGRES_PASSWORD` from env
- Healthcheck: `pg_isready -U ${DATABASE_USER}`
- Ports: `5432:5432` (external access for development)

**minio:**
- Image: `minio/minio:latest`
- Command: `server /data --console-address ":9001"`
- Volumes: `minio_data:/data`
- Environment: `MINIO_ROOT_USER`, `MINIO_ROOT_PASSWORD`
- Ports: `9000:9000` (API), `9001:9001` (Console)

**app:**
- Build from Dockerfile.prod, target: production
- Ports: `3000:3000`
- Environment: DATABASE_URL, MINIO_ENDPOINT, MINIO_ACCESS_KEY, MINIO_SECRET_KEY, PDF_STORAGE_TYPE=s3
- Depends on: postgres (healthy), minio (started)
- Healthcheck: `wget --spider http://localhost:3000/api/health`
- Healthcheck: interval=30s, timeout=10s, retries=3, start_period=60s

**caddy:**
- Image: `caddy:2-alpine`
- Ports: `80:80`, `443:443`
- Volumes: `./Caddyfile:/etc/caddy/Caddyfile`, caddy_data, caddy_config
- Depends on: app (healthy)

**Volumes:** postgres_data, minio_data, caddy_data, caddy_config
</details>
</task>

<details>
Update `.env.example` with production variables:

```bash

# MinIO/S3 Storage

MINIO_ACCESS_KEY="minioadmin"
MINIO_SECRET_KEY="minioadmin"

# Next.js

# Security
NEXTAUTH_SECRET="generate-with-openssl-rand-base64-32"
NEXTAUTH_URL="https://ai-afterschool.example.com"

# PDF Storage (fallback for local)
PDF_STORAGE_PATH="./public/reports"
```

Add README section for environment setup:
- Generate NEXTAUTH_SECRET with `openssl rand -base64 32`
- Change default MinIO credentials in production
</details>
</task>

<task>
<action>Test Docker Compose build and startup</action>
<command>docker-compose -f docker-compose.prod.yml build</command>
<details>
1. Build images: `docker-compose -f docker-compose.prod.yml build`
2. Start services: `docker-compose -f docker-compose.prod.yml up -d`
3. Verify containers running: `docker-compose -f docker-compose.prod.yml ps`
4. Check logs: `docker-compose -f docker-compose.prod.yml logs app`
5. Stop and cleanup: `docker-compose -f docker-compose.prod.yml down`

Expected result:
- All 4 services (postgres, minio, app, caddy) show as "Up"
- App container healthcheck passes after startup
- No errors in logs
</details>
</task>
