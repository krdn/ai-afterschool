<action>Create Caddyfile with reverse proxy configuration</action>
<file>Caddyfile</file>
<details>
Create `Caddyfile` with configuration:

```caddyfile
# Global options
{
    # Email for Let's Encrypt notifications
    email admin@example.com

    # Agree to Let's Encrypt terms
    on_demand_tls {
        ask http://app:3000/api/health
    }
}

# Production domain
ai-afterschool.example.com {
    reverse_proxy app:3000 {
        # Health check upstream
        health_uri /api/health
        health_interval 30s
        health_timeout 10s
    }

    # Security headers
    header {
        # Prevent clickjacking
        X-Frame-Options "SAMEORIGIN"
        # Prevent MIME type sniffing
        X-Content-Type-Options "nosniff"
        # Referrer policy
        Referrer-Policy "strict-origin-when-cross-origin"
        # XSS protection
        X-XSS-Protection "1; mode=block"
        # HSTS (uncomment after first successful SSL)
        # Strict-Transport-Security "max-age=31536000; includeSubDomains"
    }

    # Enable compression
    encode gzip zstd

    # Log access (JSON format for parsing)
    log {
        output file /var/log/caddy/access.json
        format json
    }
}

# HTTP to HTTPS redirect
http://ai-afterschool.example.com {
    redir https://ai-afterschool.example.com{uri} permanent
}

# MinIO console (optional - direct access without proxy)
:9001 {
    reverse_proxy minio:9001
}

# Development mode (localhost) - no SSL
:80 {
    reverse_proxy app:3000
    log {
        output stdout
        format console
    }
}
```

**IMPORTANT:** Replace `ai-afterschool.example.com` with actual production domain before deployment.

<task>
<action>Update docker-compose.prod.yml for Caddy configuration</action>
<file>docker-compose.prod.yml</file>
<details>
Update the `caddy` service in `docker-compose.prod.yml`:

```yaml
caddy:
  image: caddy:2-alpine
  container_name: caddy
  restart: unless-stopped
  ports:
    - "80:80"
    - "443:443"
    - "443:443/udp"  # HTTP/3
  volumes:
    - ./Caddyfile:/etc/caddy/Caddyfile:ro
    - caddy_data:/data
    - caddy_config:/config
    - caddy_logs:/var/log/caddy
  environment:
    # Email for Let's Encrypt
    ACME_AGREE: "true"
    EMAIL: "${CADDY_EMAIL:-admin@example.com}"
  depends_on:
    app:
      condition: service_healthy
  networks:
    - backend
```

Add `caddy_logs` volume to volumes section.

**IMPORTANT:** Email is required for Let's Encrypt certificate expiration notices.
</details>

## Production Deployment

### SSL/TLS Configuration

Caddy automatically obtains and renews SSL certificates from Let's Encrypt.

**Requirements:**
- Domain DNS A record pointing to server IP
- Port 80 and 443 accessible from internet
- Valid email address in Caddyfile for expiration notices

**First deployment:**
1. Update `ai-afterschool.example.com` in Caddyfile to actual domain
2. Update email in Caddyfile
3. Ensure DNS propagation is complete
4. Run `docker-compose -f docker-compose.prod.yml up -d`
5. Certificate is obtained automatically on first HTTPS request

**Troubleshooting:**
- Certificate fails: Check DNS, ensure ports 80/443 open
- Rate limited: Use Let's Encrypt staging (comment in Caddyfile)
- Certificate not renewing: Check email for notices from Let's Encrypt

**Manual certificate check:**
```bash
docker-compose -f docker-compose.prod.yml logs caddy | grep -i "certificate"
```
```

- MinIO console on port 9001 bypasses Caddy (direct access for management)
- Caddy stores certificates in `/data` volume (persists across restarts)
