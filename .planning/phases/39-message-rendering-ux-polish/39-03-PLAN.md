**1. suggestionsPlacement 변경**

`chat-input.tsx`의 `<MentionsInput>` 컴포넌트에서:

```diff
- suggestionsPlacement="above"
+ suggestionsPlacement="auto"
```

이 한 줄 변경으로 react-mentions-ts가 자동으로 뷰포트 높이를 감지하여:
- 기본적으로 위쪽에 드롭다운 표시 (기존 동작 유지)
- 뷰포트 상단 공간이 부족하면 아래쪽으로 자동 전환

react-mentions-ts 내부 로직 (39-RESEARCH.md에서 확인):
```
shouldShowAboveCaret = suggestionsPlacement === "above" ||
  (suggestionsPlacement === "auto" &&
   top + suggestions.offsetHeight > viewportHeight &&
   suggestions.offsetHeight < top - caretHeight)
```

**2. 스트리밍 중 비활성화 확인**

Phase 38에서 이미 `disabled={isStreaming}`이 `<MentionsInput>`에 적용되어 있음. 추가 변경 불필요.

코드에서 확인: `src/components/chat/chat-input.tsx` 라인 ~149:
```tsx
disabled={isStreaming}
```

이 prop은 `<MentionsInput>` 전체를 비활성화하여:
- 텍스트 입력 불가
- @ 트리거 불가 → 드롭다운 열림 방지
- 포커스 방지

**3. (선택적) 빈 결과 메시지 — 스킵 사유**

react-mentions-ts는 검색 결과가 빈 배열([])일 때 드롭다운 자체를 표시하지 않음 (라이브러리 기본 동작). 사용자 입장에서:
- @를 입력하면 최소 2자 입력 전에는 드롭다운 안 열림 (useMention에서 빈 배열 반환)
- 2자 이상 입력 후 결과 없으면 드롭다운 닫힘

"검색 결과 없음" 메시지를 표시하려면 data 함수에서 disabled placeholder 아이템을 반환해야 하지만, 이는 react-mentions-ts의 동작을 해킹하는 것이므로 현재 스킵한다. 필요시 향후 UX 개선으로 추가할 수 있음.

주석으로 의도 기록:
```typescript
// NOTE: 빈 결과 시 react-mentions-ts는 드롭다운을 표시하지 않음 (라이브러리 기본 동작)
// "검색 결과 없음" 메시지 표시가 필요하면 fetchMentions에서 placeholder 아이템 반환 패턴 적용
```
