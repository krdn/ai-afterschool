**1. parseMentionChips 유틸리티 (`src/lib/chat/parse-mention-chips.ts`)**

ContentSegment 타입과 parseMentionChips 함수를 생성한다:

```typescript
export type ContentSegment =
  | { kind: "text"; text: string }
  | { kind: "mention"; entity: MentionedEntity }
```

`parseMentionChips(content: string, entities: MentionedEntity[]): ContentSegment[]` 함수:
- `entities`가 null/undefined/빈 배열이면 `[{ kind: "text", text: content }]` 반환 (구형 메시지 호환)
- `entities`를 `displayName.length` 내림차순 정렬 (부분 매칭 방지 — "김철수 선생님"이 "김철수"보다 먼저 매칭)
- 정렬된 displayName으로 regex 빌드: `@(displayName1|displayName2|...)` (특수문자 이스케이프)
- `pattern.exec(content)` 반복으로 text/mention 세그먼트 분리
- 39-RESEARCH.md Pattern 1 예제 참고

MentionTag 구현:

- `PopoverTrigger asChild`: `<span>` 칩 렌더링
  - 엔티티 타입별 색상:

  - 내용: `@{entity.displayName}`
  - accessDenied인 경우: `opacity-50 cursor-not-allowed line-through` 추가, Popover 비활성화

**Team:**
```typescript
const team = await db.team.findFirst({
  where: {
    id,

// 반환: { name, sublabel: `학생 ${_count.students}명 · 교사 ${_count.teachers}명`, summary: null }
```

Team RBAC 주의: Team의 경우 `session.teamId`와 Team.id를 직접 비교하기보다, 자신의 team에 속한 학생/교사가 있는 팀인지 확인한다. 가장 간단한 방법: `session.role !== 'DIRECTOR'` 이면 `id === session.teamId` 조건만 적용 (자기 팀만 조회 가능). 더 간결하게: `where: { id, ...(session.role !== 'DIRECTOR' ? { id: session.teamId } : {}) }` — 단, id가 두 번 들어가므로 `{ id }` + `AND: [{ id: session.teamId }]` 구조 또는 단순히 non-DIRECTOR는 teamId 체크.

간소화: Team 검색 시 `session.role !== 'DIRECTOR'`이면 `where: { id, id: session.teamId ?? 'no-match' }` 대신 별도 검증:
```typescript
if (session.role !== 'DIRECTOR' && id !== session.teamId) {
  return NextResponse.json({ error: 'Not found' }, { status: 404 })
}
const team = await db.team.findUnique({ where: { id }, ... })
```

응답 타입: `{ name: string, sublabel: string, summary: string | null }`

에러 응답:
- 400: 잘못된 파라미터
- 401: 미인증 (verifySession이 처리)
- 404: 엔티티 미발견 또는 RBAC 실패
