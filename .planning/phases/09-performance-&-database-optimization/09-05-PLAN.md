       ```tsx
       // Cloudinary URL에서 publicId 추출
       function extractPublicId(url: string): string {
         // 예: https://res.cloudinary.com/.../image/upload/v123456/students/student123/profile/abc123
         // → students/student123/profile/abc123
         const match = url.match(/\/upload\/v\d+\/(.+)$/)
         return match ? match[1] : url
       }
       ```

    - `width`, `height`는 필수 prop (Next.js Image 요구사항)
    - `sizes` prop을 추가하여 반응형 최적화

  <name>Task 2: CldImage 속성 최적화</name>
  <files>src/components/students/student-image-uploader.tsx</files>
  <action>
    CldImage 컴포넌트의 속성을 최적화하여 성능과 화질 균형 달성:

    1. 이미지 품질 설정:
       ```tsx
       <CldImage
         width={128}
         height={128}
         src={publicId}
         sizes="128px"
         alt={`${label} 미리보기`}
         className="h-32 w-32 rounded-md object-cover"
         crop="fill"
         gravity="auto"
         quality="auto"  // 자동 품질 (파일 크기와 화질 균형)
         format="auto"   // 자동 형식 선택 (WebP/AVIF)
       />
       ```

    2. 반응형 sizes 설정:
       ```tsx
       sizes="(max-width: 768px) 100vw, 128px"
       ```
       - 모바일: 전체 너비
       - 데스크톱: 128px 고정

    3. 이미지 변환 옵션:
       ```tsx
       // Cloudinary 변환 파라미터
       // - crop="fill": 지정된 크기에 맞춰 채우기
       // - gravity="auto": 주요 피사체 중심으로 자르기
       // - quality="auto": 파일 크기와 화질 자동 최적화
       // - format="auto": 브라우저 지원 형식 자동 선택 (WebP/AVIF)
       ```

    4. 레이지 로딩:
       - CldImage는 기본적으로 레이지 로딩 활성화
       - `loading` prop을 명시적으로 설정 가능:
       ```tsx
       <CldImage
         // ...
         loading="lazy"  // 기본값
       />
       ```

    5. 우선순위 힌트 (첫 번째 이미지):
       - 첫 번째 이미지에 `priority` prop 추가 가능:
       ```tsx
       <CldImage
         // ...
         priority={false}  // 썸네일에는 false 권장
       />
       ```

    **주의사항:**
    - CONTEXT.md: "이미지 품질-크기 트레이드오프는 균형 있는 품질 (60-80)" → `quality="auto"` 사용
    - 썸네일 크기(128px)에서는 높은 품질 불필요
    - `crop="fill"`과 `gravity="auto"` 조합으로 이미지 비율 유지

    **RESEARCH.md 참고:**
    - Open Question 3: "Image Thumbnail Strategy - Use CldImage with responsive sizing"
