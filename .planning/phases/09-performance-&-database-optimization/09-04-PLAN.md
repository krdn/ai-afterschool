<task type="auto">
  <name>Task 2: Prisma Schema에 복합 인덱스 추가</name>
  <files>prisma/schema.prisma</files>
  <action>
    식별된 쿼리 패턴을 기반으로 Prisma schema에 복합 인덱스 추가:

    1. Student 모델에 복합 인덱스 추가:
       ```prisma
       model Student {
         // ... 기존 필드 ...

         // 기존 인덱스 (유지)
         @@index([teacherId])
         @@index([name])
         @@index([school])

         // 새로운 복합 인덱스
         @@index([teacherId, name])        // 학생 검색 (교사 내 이름 검색)
         @@index([teacherId, school])      // 학교별 필터링
         @@index([expiresAt])              // 만료된 학생 정리 (배치 작업)
         @@index([calculationRecalculationNeeded]) // 재계산 필요 학생
       }
       ```

    2. ReportPDF 모델에 status 인덱스 추가:
       ```prisma
       model ReportPDF {
         // ... 기존 필드 ...

         @@index([studentId])  // 기존 (유지)
         @@index([status])     // 새로운: PDF 생성 상태 모니터링
       }
       ```

    **인덱스 설계 원칙:**
    - `teacherId`가 포함된 복합 인덱스: teacherId를 먼저 배치 (선택도 높음)
    - 단일 컬럼 인덱스보다 복합 인덱스 우선 (인덱스 병합 방지)
    - 정렬이 필요한 경우 컬럼 순서 고려

    **주의사항:**
    - 기존 단일 인덱스(teacherId, name, school)는 제거하지 않음 (다른 쿼리에서 사용 가능)
    - 인덱스 이름은 Prisma가 자동 생성 (예: `Student_teacherId_name_idx`)
    - 쓰기 성능 저하를 고려하여 최소한의 인덱스만 추가

    **RESEARCH.md 참고:**
    - Code Examples: Database Indexes
    - Pattern 4: Index verification with EXPLAIN ANALYZE
  </action>

       - `CREATE INDEX` 문이 포함되어 있는지 확인

    2. 로컬 데이터베이스에 마이그레이션 적용:
       ```bash
       npx prisma migrate deploy
       ```

    3. EXPLAIN ANALYZE로 인덱스 사용 확인:
       ```sql
       -- PostgreSQL 쉸에서 실행 (psql 또는 Prisma Studio)
       EXPLAIN ANALYZE
       SELECT * FROM "Student"
       WHERE "teacherId" = 'teacher_123'
       AND "name" LIKE '김%';

       -- 예상 출력:
       -- Index Scan using Student_teacherId_name_idx on Student (cost=0.00..10.00 rows=10)
       -- (actual time=0.05..0.15 rows=25 loops=1)
       ```

    4. 인덱스 미사용 시 확인:
       - "Seq Scan" (전체 테이블 스캔) → 인덱스가 사용되지 않음
       - 원인: 통계 정보 부족, 쿼리 플래너 판단, 인덱스 컬럼 순서 등

    5. 인덱스 사용 확인 방법:
       ```bash
       # Prisma format + migrate deploy로 재생성
       npx prisma format
       npx prisma migrate deploy

       # PostgreSQL 통계 정보 갱신
       psql -c "ANALYZE;"
       ```

    **주의사항:**

    - CONTEXT.md: "데이터베이스 인덱스는 실제로 느린 쿼리(EXPLAIN 분석)에서만 추가"
