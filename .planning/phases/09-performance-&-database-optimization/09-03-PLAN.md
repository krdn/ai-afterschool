  <action>
    식별된 N+1 패턴을 `include`를 사용하여 단일 쿼리로 최적화:

    **최적화 패턴 (personality-summary.ts 참고):**
    ```typescript
    const student = await db.student.findFirst({
      where: {
        id: params.id,
        teacherId: session.userId,
      },
      include: {
        images: true,                    // StudentImage 관계
        sajuAnalysis: true,              // SajuAnalysis 관계
        nameAnalysis: true,              // NameAnalysis 관계
        mbtiAnalysis: true,              // MbtiAnalysis 관계
        faceAnalysis: true,              // FaceAnalysis 관계
        palmAnalysis: true,              // PalmAnalysis 관계
        personalitySummary: true,        // PersonalitySummary 관계
        reportPDF: true,                 // ReportPDF 관계
      },
    })

    if (!student) {
      notFound()
    }
    ```

    **적용 단계:**
    1. 기존 코드에서 별도 조회를 모두 제거
    2. 단일 `findFirst` 호출에 모든 관계를 `include`로 추가
    3. 컴포넌트에서 `student.images`, `student.sajuAnalysis` 등으로 직접 접근
    4. 불필요한 서버 액션/데이터 페칭 함수 제거

    **데이터 전달:**
    - Server Component props로 데이터 전달
    - 예: `<StudentDetail student={student} />`

    **주의사항:**
    - `teacherId` 검증을 유지 (보안)
    - 모든 관계가 Prisma schema에 정의되어 있는지 확인
    - `select`를 사용하는 경우 `include`와 함께 사용 불가 → `include`만 사용
    - 필요한 관계만 include (불필요한 조인 방지)

    **검증:**
    - 쿼리 로그에 단일 `SELECT` 문만 표시되어야 함
    - 관계가 JOIN 또는 LATERAL JOIN으로 로드되는지 확인
